## Problem Statement

A logistics company operates an extensive delivery network spanning multiple cities connected by roads with varying fuel costs. The engineering team needs to optimize their infrastructure investments by analyzing the minimum spanning tree (MST) of their delivery network and efficiently processing numerous strategic queries about potential route modifications.

You are tasked with implementing a sophisticated offline MST query system that pre-computes the minimum spanning tree and handles complex analytical queries about network modifications, route criticality, and cost optimization. The system must process large datasets of cities and roads while providing instant responses to infrastructure planning queries.

## Core Requirements

Your solution must implement a **pure Python** offline MST query algorithm (no external libraries) that processes delivery network data and efficiently answers strategic planning queries. The algorithm should pre-compute necessary data structures to enable fast query responses for infrastructure decision-making.

### Key Operational Constraints

1. **MST Construction Constraints**: Build the minimum spanning tree using Kruskal's algorithm with Union-Find data structure
2. **Edge Weight Constraints**: All road costs must be positive integers representing fuel consumption units
3. **Connectivity Constraints**: The network must remain connected after any edge removal queries
4. **Memory Efficiency Constraints**: Preprocessing data structures must not exceed O(n²) space complexity

### Advanced Algorithmic Requirements

The offline MST query system must implement a multi-faceted analysis framework that evaluates network modifications using these computational methods:

**Critical Edge Analysis**: For each edge in the MST, pre-compute the maximum weight edge that would be added if this edge were removed. This enables instant "what-if" analysis for infrastructure maintenance planning.

**Bottleneck Path Queries**: For any two cities, determine the maximum cost edge along the unique path in the MST. This represents the most expensive segment that limits efficient delivery between these locations.

**Edge Addition Impact**: Calculate cost reduction by finding the maximum weight edge in the unique cycle created by adding the new edge to the current MST. If the new edge cost is less than the maximum cycle edge cost, cost_reduction = max_cycle_edge_cost - new_fuel_cost, otherwise cost_reduction = 0. Calculate efficiency_gain = (cost_reduction / total_mst_cost) * 100

**Network Vulnerability Assessment**: Identify edges whose removal would cause the maximum increase in total MST cost, representing critical infrastructure that requires protection and redundancy planning.

The algorithm must also implement cost optimization analysis, where edges with weight greater than the 75th percentile of all edge weights are classified as "high-cost" segments requiring priority review for alternative routing solutions.

**Cost Classification Formula**:
- Calculate Q3 = 75th percentile of all edge weights
- If edge_weight > Q3: classify as "high_cost_segment"
- Otherwise: classify as "standard_cost_segment"

## Implementation Guidelines

**MST Construction Strategy**: Implement Kruskal's algorithm with Union-Find data structure for efficient MST computation with O(E log E) time complexity

**Preprocessing Phase**: Build auxiliary data structures including parent arrays for LCA queries, edge replacement tables, and bottleneck matrices for O(1) query responses

**Critical Edge Analysis**: For each MST edge, pre-compute the minimum cost replacement edge by finding the maximum weight edge in the cycle formed when adding each non-MST edge. Use `critical_impact_threshold` and `medium_impact_threshold` from cost_threshold to classify impact severity (high: replacement_cost > original_cost * critical_impact_threshold, medium: replacement_cost > original_cost * medium_impact_threshold, low: otherwise).

**Bottleneck Path Computation**: Use Lowest Common Ancestor (LCA) preprocessing to enable O(log n) bottleneck queries between any two cities

**Edge Addition Impact**: Calculate cost reduction by finding the maximum weight edge in the unique cycle created by adding the new edge to the current MST

**Vulnerability Assessment**: Rank MST edges by their removal impact on total network cost and identify critical infrastructure requiring redundancy

**Cost Classification**: Apply the 75th percentile threshold to classify edges and prioritize high-cost segments for optimization review. Use max_acceptable_cost to filter roads during preprocessing and high_cost_multiplier to identify critical segments.

**Network Statistics**: Calculate connectivity metrics, average path costs, and identify top `congestion_analysis_count` bottleneck segments based on capacity and alternative routing availability. For redundancy classification, use edges with cost > (75th_percentile * redundancy_cost_factor).

**Deterministic Processing**: Ensure consistent results by processing edges and queries in sorted order by their identifiers

**Memory Optimization**: Use space-efficient data structures and avoid redundant storage while maintaining O(log n) query performance
