Problem Statement:
### Problem Statement
You are tasked with implementing a circulant matrix decomposition algorithm for network adjacency matrices. A **circulant matrix** is a special matrix where each row is a cyclic right shift of the previous row. 

Given an adjacency matrix representing a network with weighted edges, decompose it into a sum of circulant matrices. The decomposition must satisfy: **Input Matrix = Sum of all Output Circulant Matrices**.

### Key Definitions:

**Circulant Matrix Property:**
For a circulant matrix C with first row [c₀, c₁, c₂, ..., cₙ₋₁]:
- Row 0: [c₀, c₁, c₂, ..., cₙ₋₁]
- Row 1: [cₙ₋₁, c₀, c₁, ..., cₙ₋₂]
- Row 2: [cₙ₋₂, cₙ₋₁, c₀, ..., cₙ₋₃]
- And so on (each row is a right cyclic shift)

**Decomposition Requirement:**
If A is the input matrix and C₁, C₂, ..., Cₖ are the output circulant matrices, then:
A[i][j] = C₁[i][j] + C₂[i][j] + ... + Cₖ[i][j] for all positions (i, j)

### Algorithm Specification:

The algorithm works by iteratively extracting circulant components:

1. **Initialization:** Start with the input matrix as the "remaining" matrix.

2. **Circulant Extraction:** For each iteration:
   - Select a non-zero element at position (r, c) in the remaining matrix with value v.
   - Create a circulant matrix where position (i, j) has value v if (j - i) mod n == (c - r) mod n, else 0.
   - This ensures the circulant pattern contains value v at position (r, c).
   - Subtract this circulant from the remaining matrix (element-wise).

3. **Termination:** Continue until the remaining matrix contains only zeros.

4. **Best Match Criteria:** When multiple circulants are possible, select the one that:
   - Maximizes the number of positions where the circulant value matches the remaining matrix value.
   - This greedy approach minimizes the total number of circulant matrices needed.
