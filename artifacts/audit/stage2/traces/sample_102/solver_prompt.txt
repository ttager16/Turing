## Problem Statement
A logistics company operates a sophisticated delivery network where the assignment of delivery trucks to routes must consider both operational efficiency and sequential dependencies between routes. The company's network is modeled as a bipartite graph where trucks and routes form two distinct sets, with additional constraints representing route dependencies that must be satisfied for operational compliance.

You are tasked with implementing an advanced Dependent Matching algorithm that finds optimal truck-to-route assignments while respecting all dependency constraints and operational limitations. The system must efficiently process complex routing scenarios where certain routes can only be executed after specific prerequisite routes are completed, creating a hierarchical dependency structure that affects the overall matching strategy.

## Core Requirements
Your solution must implement a **pure Python** Dependent Matching algorithm (no external libraries) that processes logistics network data and generates optimal truck-route assignments. 

### Key Operational Constraints
1. **Dependency Constraints**: Routes may have prerequisite routes that must be completed before execution
2. **Capacity Constraints**: Each truck has a maximum load capacity and route coverage limitations
3. **Time Window Constraints**: Routes must be completed within specified operational time frames
4. **Priority Level Constraints**: Higher priority routes (lower priority numbers) must be assigned first
5. **Resource Availability**: Trucks have operational hour limits and maintenance requirements


### Advanced Algorithmic Requirements
The Dependent Matching algorithm must incorporate a multi-criteria optimization system that evaluates assignment viability using this composite formula:
**Assignment Efficiency Score = Base Assignment Cost + Capacity Penalty + Dependency Penalty + Time Penalty + Window Urgency Penalty**

**Optimization Objective**: The algorithm must **minimize** the Assignment Efficiency Score to find optimal assignments (lower scores indicate better assignments).

Where:
- Base Assignment Cost: The base_assignment_cost value from the route specification
- Capacity Penalty: Calculated as (load_requirement / truck_capacity) × 100 when load_requirement > truck_capacity × 0.8, otherwise 0
- Dependency Penalty: Number of unresolved prerequisite routes for this route × 50
- Time Penalty: Calculated as (scheduled_start_time - end_time_window) × 10 when scheduled_start_time > end_time_window, otherwise 0
- Window Urgency Penalty: Calculated as 200 / (end_time_window - start_time_window + 1) (higher urgency for tighter time windows)

**Total Assignment Cost Calculation**:
The total assignment cost includes base assignment cost (with dependency adjustment first, then priority weight factor), distance-based fuel cost, operational hour cost, and overtime cost when applicable.

**Cost Formula Components** (applied in this order):
1. Dependency-adjusted base cost: `base_assignment_cost × (1 + 0.15 × unresolved_dependencies)` if unresolved_dependencies > 0, otherwise `base_assignment_cost`
2. Priority-adjusted base cost: `dependency_adjusted_base_cost × priority_weight_factor`
3. Fuel cost: `Manhattan_distance × fuel_cost_per_km`
4. Hour cost: `base_cost_per_hour × actual_duration_with_complexity`
5. Overtime cost: Applied when total duration exceeds operational_hours

**Duration with Complexity Calculation**:
`actual_duration_with_complexity = estimated_duration × route_complexity`

**Overtime Cost Formula**:
When total duration exceeds operational_hours:
`overtime_cost = overtime_hours × base_cost_per_hour × (overtime_multiplier - 1)`
Where `overtime_hours = total_duration - operational_hours`

**trucks**: List of dictionaries, each containing:

**routes**: List of dictionaries, each containing:

- minimum_gap_hours: Required time gap between prerequisite completion and dependent start (integer, 0-8)
**operational_constraints**: Dictionary containing:
- max_routes_per_truck: Maximum number of routes assignable to one truck (integer, 1-10)
- fuel_cost_per_km: Cost calculation factor for distance-based pricing (float, 0.5-2.0)
