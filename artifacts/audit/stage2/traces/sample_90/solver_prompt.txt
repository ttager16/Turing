### Problem Statement

In the high-frequency trading sector, a Python developer is tasked with designing a next-generation real-time analytics tool to manage and analyze a dynamic portfolio of stock symbols. Unlike traditional single-stream monitoring, this system must handle compound queries across multiple symbols, each with continuously fluctuating stock prices and various aggregation needs. The solution should employ advanced data structures (e.g., Segment Trees or Treaps) to concurrently track minimum and maximum prices per symbol, efficiently compute partial sums, and perform snapshot merges of related symbols-all while maintaining strict time and memory efficiency to accommodate millions of operations. By integrating multi-step reasoning and stateful interactions among different symbols, the developer must ensure minimal latency and maximal throughput under high-volume trading conditions.


#### Objectives

1. Create a unified data structure capable of:
   * Inserting a stock price for a specific symbol in O(log n) or better time.
   * Removing the most recent price while preserving aggregated historical data.
   * Providing real-time minimum and maximum stock prices across individual symbols, as well as globally.
   * Performing snapshot merges of related symbols to derive extended market insights.
2. Support multi-dimensional queries, including compound checks (e.g., finding the min of one symbol while simultaneously computing the max of another), and interval-based aggregations (e.g., partial sum).
3. Design the solution to handle complex corner cases such as interleaved updates across symbols, disjoint symbol sets, and simultaneous push/pop bursts.
4. Optimize memory usage and ensure determinism across operations, without using brute-force recomputation.


#### Constraints

- The solution must be implemented in pure Python (no external libraries allowed).
- Stock prices are integers in the range [-10^9, 10^9].
- Operations include insertion, removal, multi-symbol min/max queries, range-based computations, and cluster merges.
- Deterministic routing policy:
  * The active symbol starts at ID 0.
  * ['merge_clusters', None] creates a new symbol (next available ID) as a deep snapshot of the active symbol but does not switch the active symbol.
  * The first subsequent ['push', price] after a merge is routed to the new symbol (one-shot redirect).
  * ['pop', None] targets the most recently pushed-to symbol (MRU). if none, use the active symbol.
- Deep snapshot semantics: After merge, source and new symbols are independent, subsequent pushes/pops to one must not affect the other.
- Multiple occurrences of a symbol’s min/max must be handled so that popping one occurrence does not change that symbol’s get_min (or get_max) until the last occurrence is removed.
- Global get_min(None) / get_max(None) must reflect the exact current state over all symbols. If lazy heaps are used, stale entries must not leak into answers.
- ['range_sum', s] returns the sum of values that existed in symbol s at the time of its most recent merge event involving s, if s has never merged, return the sum of all values ever pushed so far to s. This baseline is analytic-only and does not decrease due to later pops.

- Each operation must be a list of exactly two elements, e.g., ['push', 100] or ['get_max', None].
