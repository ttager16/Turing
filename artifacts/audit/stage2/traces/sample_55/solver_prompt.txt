### Problem Statement

A financial analytics firm requires a Python-based algorithm to efficiently process and rank a large network of multi-level investment portfolios. Each portfolio may contain nested sub-portfolios and cross-references to other portfolios, forming an interconnected dependency graph. Every portfolio includes measurable attributes such as risk and return values, and the algorithm must determine a globally consistent ranking based on these metrics.

The goal is to design a deterministic and validation-strict sorting system that can handle complex data hierarchies, cyclical references, and missing relationships without compromising structural integrity or reproducibility. The function must compute composite scores derived from return-to-risk ratios and cross-reference influence, sort top-level portfolios by these scores, and preserve the nested structure of all sub-portfolios. Any malformed or invalid input must trigger fail-fast behavior by returning an empty list immediately.


### Objectives

1. Graph Construction and Validation
   * Construct a directed graph from portfolios, capturing both sub-portfolio (hierarchical) and reference (cross-link) edges.
   * Validate input structure before computation. Any invalid node or field must cause an immediate [] return.
   * All adjacency relationships must be deduplicated and sorted lexicographically.
2. Cycle Handling and Deterministic Scoring
   * Detect strongly connected components (cycles).
   * All nodes within a cycle share the same final score, equal to the arithmetic mean of their members base scores (return / risk). This value is not further adjusted by reference averaging for those members and is used only when other portfolios average over their references.
   * Within a cycle, the lexicographically smallest portfolio name represents the group for ordering.
3. Score Calculation
   For each portfolio p:
   score(p) = (return / risk) + 0.1 * average(score(references))
   * References to missing portfolios are ignored.
   * All arithmetic must remain finite.
   * No division by zero is allowed.
4. Stable Sorting and Output Preservation
   * Sort only top-level portfolios by:
     1. Descending _score (higher first)
     2. Ascending name (for ties)
   * Preserve the internal order of sub_portfolios and references exactly as in the input.
5. Deterministic Traversal Rules
   * All nodes and adjacency lists must be processed in lexicographic order.
   * Traversal or set iteration must never depend on Pythons internal hashing order.
   * Lexicographic processing and deduplication apply to internal graph structures only and must not modify the order of references or sub_portfolios in the returned output.
