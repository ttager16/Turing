### Problem Statement

In a rapidly evolving multi-tier real-time bidding (RTB) ecosystem, the marketing firm must orchestrate ad placement across a complex network of audience segments, each interconnected by synergy constraints and dynamic ROI implications. A fixed advertising budget is shared among multiple parallel campaigns, where each bid opportunity pertains to specific user segments and offers varying returns.

Unlike simple greedy approaches, the challenge involves tracking segment overlap, heterogeneous synergy effects, and ensuring the best overall allocation of the limited budget.

Within this environment, you must design an algorithmic infrastructure capable of:

1. Maintaining a representation of all segment interconnections.
2. Computing feasible ad placements under synergy and budget constraints.
3. Maximizing the net impressions gained while respecting budget and ROI limits.
4. Applying greedy selection **with local improvements**, iteratively refining the allocation through profitable additions and swaps until no further gain is possible.

This requires employing efficient Python data structures (e.g., dictionaries, sets, or lists) and heuristic optimization strategies such as greedy selection followed by local repair steps. A naive brute-force approach fails to scale, while advanced but lightweight heuristics can achieve near-optimal results.

---

### Objectives

1. **Segmented Graph Construction:** Build and maintain relationships between user segments, each carrying distinct cost and ROI values.
2. **Synergy-Aware Optimization:** Incorporate synergy penalties between overlapping segments when deciding which bids to accept.
3. **Budget-Constrained Allocation:** Ensure that total spending never exceeds the available budget.
4. **ROI Maximization:** Select bids that maximize overall utility (ROI minus penalties) while staying within all limits.

---

### Constraints

* Implement the solution using **pure Python only** — no external libraries are allowed.
* Each bid may cover one or more segments.
* **Synergy effects are pairwise and presence-based**:

  * For any unordered pair of segments *(a, b)*, if both appear in the accepted set (even across different bids), apply the synergy penalty **exactly once**.
  * Self-pairs *(a, a)* are ignored.
* **ROI allocation** is evenly split across all segments in a bid.

  * If a segment has a cap, only its remaining capacity contributes to ROI; excess is discarded.
  * Segments without caps are treated as unlimited.
* The total cost of selected bids cannot exceed the budget.
* The optimizer must perform **local improvement steps** after the initial greedy phase, adding or swapping bids that increase total utility without violating constraints, until a stable configuration is reached.
* The algorithm must be **deterministic**: identical inputs must yield identical outputs.
* Utility is computed as:

  ```
  utility = total_roi_after_caps - total_penalty
  ```
* Total bids ≤ 10,000; segments per bid ≤ 4.
