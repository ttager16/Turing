## Problem Statement

Develop a comprehensive Python-based electric vehicle (EV) charging schedule optimizer for smart city infrastructure. This system serves as a critical component for urban mobility platforms that require real-time allocation of EVs to charging stations while minimizing wait times, energy consumption, and grid load. The system must implement advanced graph-based optimization algorithms including min-cost flow with Johnson's reweighting technique and segment trees for efficient capacity tracking, enabling dynamic scheduling that adapts to real-time traffic patterns, EV battery levels, and station availability constraints.

## Problem Description

The system optimizes EV charging station assignments using a combination of min-cost flow optimization and priority-based scheduling. The framework must implement:

### Core Features

1. **Traffic-Aware Routing**: Calculate travel time based on geographic distance and real-time congestion levels
2. **Min-Cost Flow Optimization**: Implement successive shortest paths algorithm using Dijkstra with potential functions
3. **Johnson's Reweighting**: Use Bellman-Ford for initial potential computation to handle negative edge costs
4. **Segment Tree Capacity Tracking**: Efficient O(log n) range queries and updates for station availability over time
5. **Priority-Based Scheduling**: Prioritize EVs by battery level with immediate priority for critical (<20%) levels
6. **Peak Hour Restrictions**: Apply capacity constraints (60% limit) when congestion exceeds threshold
7. **Dynamic Cost Calculation**: Adjust costs based on station load and capacity utilization ratios
8. **Iterative Refinement**: Perform flow-based assignment with post-optimization for better solutions
9. **Time Slot Allocation**: Find earliest available charging slots using segment tree queries
10. **Real-time Constraint Validation**: Check feasibility of EV-station assignments before graph construction

### Algorithm Components

The implementation requires three main algorithmic components:

**FlowGraph Class**: Implements min-cost flow using successive shortest paths
- Store edges in defaultdict(list), capacity/cost/flow in dictionaries keyed by (u,v) tuples
- For each edge (u,v) with capacity cap and cost c, also create reverse edge (v,u) with capacity 0 and cost -c
- Skip adding edges if capacity <= 0
- Bellman-Ford initializes potentials by relaxing edges up to V-1 times with early termination if no updates
- Dijkstra uses heap with (distance, node) tuples and visited set to avoid reprocessing nodes
- After finding shortest path, update potentials: `potential[node] += dist[node]` for all reachable nodes
- Augment flow along path by minimum residual capacity, update flow on forward and reverse edges

**SegmentTree Class**: Tracks station capacity availability over time windows
- Build segment tree recursively storing minimum values in each node
- Lazy propagation defers updates by storing pending values in lazy array
- Update range by adding value to all elements in [l, r]; query minimum in [l, r]
- Handle empty tree case (n=0) by returning early from operations
- Query returns `float('inf')` when no valid range overlaps

**EVChargingOptimizer Class**: Orchestrates the optimization process
- Initialize traffic_map, station_map, ev_map, station_segment_trees, station_occupancy dictionaries
- Build flow graph only including EV nodes that have at least one feasible nearby station
- Extract assignments by processing EVs in priority order (pop from priority queue)
- Find earliest available slot by iterating from arrival_time, checking segment tree capacity
- Update segment tree with -1 when assigning, +1 when removing assignments during refinement
- During refinement, temporarily remove assignment to test alternatives, then restore if not better
