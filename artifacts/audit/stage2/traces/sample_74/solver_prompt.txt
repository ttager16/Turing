## Constraints

1. **Libraries**: Only standard Python typing library allowed
2. **AVLNode Attributes**: Store timestamp, list of transaction tuples, left/right child pointers, and height
3. **Transaction Tuples**: Store pairs of canonical ID and appearance index within each AVLNode
4. **TransactionMetadata Class**: Track timestamp, appearance index, and amount for each unique transaction
5. **Hash Table Components**: Maintain hash index mapping canonical IDs to metadata, AVL tree for temporal queries, and appearance counter
6. **AVL Tree Property**: Maintain binary search tree ordering by timestamp with self-balancing via rotations
7. **Balance Maintenance**: Perform rotations when balance factor (left height minus right height) exceeds ±1 after insertion
8. **Rotation Cases**: Apply right rotation for left-left case (balance > 1, new timestamp < left child), left rotation for right-right case (balance < -1, new timestamp > right child), left-right rotation for left-right case (balance > 1, new timestamp > left child), right-left rotation for right-left case (balance < -1, new timestamp < right child)
9. **Height Calculation**: Node height is one plus maximum of children heights, treating None as zero
10. **Canonical ID Parsing**: Extract portion after first '|' delimiter for IDs starting with 'B' containing '|'; otherwise use full ID (use maxsplit of 1)
11. **Branch Identification**: Detect branch prefix when transaction ID starts with 'B' and contains '|'; branch ID (string) is portion before first '|'
12. **Branch Statistics Counting**: Map branch ID strings to transaction counts; count all transactions with each branch prefix including duplicates, not just unique ones
13. **Duplicate Source Tracking**: Maintain count of how many duplicate transaction IDs were encountered
14. **Deduplication Logic**: Reject transaction insertion when canonical ID exists in hash index
15. **Appearance Ordering**: Assign incrementing sequential index to each successfully inserted transaction starting from 0
16. **Range Query Traversal**: Explore tree nodes recursively, pruning branches outside query range, collecting matching transactions
17. **Query Result Sorting**: Order range query results by appearance index before returning canonical IDs
18. **Invalid Range Handling**: Return empty list when query start exceeds end
19. **Merge Efficiency Metric**: Percentage of unique transactions from total, rounded to 2 decimals (0.0 if no transactions)
20. **Timestamp Statistics Scope**: Calculate min, max, range, and unique count only from successfully inserted transactions; min/max are None if no successful insertions
21. **Out-of-Order Detection**: Measure percentage of transactions arriving with smaller timestamp than predecessor, rounded to 2 decimals
22. **Amount Aggregation Scope**: Compute statistics only for unique successfully inserted transactions
23. **Standard Deviation Formula**: Use population variance (sum of squared deviations divided by count), rounded to 2 decimals
24. **Per-Query Metrics Structure**: Include query range, result count, and amount statistics (total, average, min, max) for each query
25. **Empty Query Response**: Set count to 0 and all financial metrics to 0.0 when query returns no results
26. **Aggregate Query Metrics**: Track total queries, sum of all results, average results per query, and empty query count
27. **Tree Statistics Components**: Report height, node count, maximum balance factor, balance status, and leaf count
28. **Empty Tree Handling**: Return zeroed statistics with is_balanced as True when tree has no root
29. **Balance Factor for Statistics**: At each node, compute absolute difference between subtree heights, track global maximum
30. **Leaf Node Definition**: Nodes with both left and right children as None
31. **Theoretical Minimum Height**: Integer square root of node count for comparison with actual height
32. **Balance Efficiency Metric**: Computed as (1 - max_balance/10) × 100, rounded to 2 decimals (100.0 when empty)
33. **Performance Rate Calculations**: Insertion success and deduplication rates as percentages of total transactions, rounded to 2 decimals
34. **Same Timestamp Handling**: Append multiple transactions to single AVL node when timestamps match, avoiding duplicate nodes
35. **Empty Input Handling**: Handle edge cases of empty transaction or query lists gracefully with appropriate default values
