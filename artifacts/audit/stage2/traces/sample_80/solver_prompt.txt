### Rules and Processing Logic

1. **Stale Data Rule:**
   Each device has a defined “freshness window” (Δ_stale = 1).
   A log `[device_id, timestamp, payload]` is considered **stale** if a newer log exists for the same device such that:

   ```
   newer_timestamp - timestamp > Δ_stale
   ```

   Stale logs must be discarded.

2. **Duplicate Entry Rule:**
   Any log with a payload exactly equal to `"duplicate_entry"` must be removed.
   This is an explicit rule and not dependent on comparison with other entries.

3. **Reference Validation Rule:**
   Some payloads reference other logs using the format:
   `"REF_<device_id>_<timestamp>"`
   Such a log is valid only if there exists a corresponding log `[device_id, timestamp, ...]` in the current dataset (after applying the stale and duplicate rules).
   Invalid or malformed references must be discarded.

4. **Order Preservation Rule:**
   The final output must preserve the original order of valid logs. No sorting or reordering is allowed.
