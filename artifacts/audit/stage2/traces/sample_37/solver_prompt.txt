## Problem Statement

Develop a comprehensive Python-based correlated asset trading simulator that leverages Geometric Brownian Motion (GBM) for stochastic price modeling and recursive backtracking for strategy optimization. This framework serves as a critical component for quantitative trading platforms that require exploration of multiple trading paths under correlated asset dynamics. The system must implement recursive divide-and-conquer algorithms with memoization to efficiently explore trading strategies, manage capital constraints, account for transaction costs, and identify optimal trading sequences that maximize portfolio value across discrete time horizons.

## Problem Description

The system simulates multi-asset trading scenarios where asset prices evolve according to correlated GBM processes. The framework must implement:

### Core Features

1. **Correlated Price Simulation**: GBM-based price evolution with correlation-adjusted drift
2. **Recursive Backtracking**: Explore multiple trading paths using divide-and-conquer strategy
3. **Dynamic Programming Memoization**: Cache explored states to avoid redundant computation
4. **Transaction Cost Modeling**: Proportional costs for buy and sell operations
5. **Capital Constraint Enforcement**: Validate trades against available capital limits
6. **Position Management**: Track long positions across multiple assets and time steps
7. **Trade Generation**: Deterministic generation of correlation-based trading options
8. **Portfolio Valuation**: Calculate final portfolio value including positions and remaining capital
9. **Strategy Explanation**: Generate human-readable descriptions of trading strategies

### Required Dictionary Keys

The main function must return a list of dictionaries, each with these exact keys:
- `path_id`: Unique identifier for the trading path (0-indexed)
- `trades`: List of trade lists for each time step
- `flow_cost`: Total transaction costs incurred
- `final_portfolio_value`: Final portfolio value including capital and positions
- `optimal_strategy_explanation`: Human-readable strategy description

5. **Base Parameters**: Use drift=0.0001, volatility=0.02, dt=1.0 for all assets
6. **Correlation Adjustment**: Calculate price_change_ratio = (current_price/initial_price - 1.0), then add correlation*price_change_ratio*0.1 to drift

8. **Deterministic Shock Generation**: For each asset at time_step, compute deterministic_value = sin(seed*0.1 + time_step*2.5 + asset_idx*3.7) * cos(time_step*1.3 + asset_idx*2.1), then z = deterministic_value * 1.5
9. **Price Iteration Order**: Iterate through assets in sorted order (self.assets) when simulating prices, using enumerate to get asset_idx for deterministic shock calculation
10. **Transaction Costs**: Buy cost = quantity*price*(1+factor), Sell proceeds = abs(quantity)*price*(1-factor); track flow_cost by adding cost*factor for buys and abs(proceeds)*factor for sells
11. **Trade Validation**: Execute buy only if cost <= capital, execute sell only if position >= abs(quantity)
12. **Quantity Rounding**: Round all trade quantities to 2 decimal places in generation
13. **Memoization Key**: Use (time_step, tuple(sorted positions rounded to 1dp), capital rounded to 0dp)
14. **Memoization Logic**: Store state only if key not exists or new final_value is better
15. **Trade Options Order**: Always include empty option first in trade options list; return max 5 options total

17. **Buy Option Logic**: Generate if asset has correlations, avg_correlation>0.5, max_buy=min(10.0, capital/(price*2))>0.1
18. **Rebalance Logic**: If time_step<time_steps//2 and positions exist, target 5.0 for first 2 assets with abs(diff)>0.1
19. **Pairs Trading**: If time_step<time_steps//3, find first asset pair with correlation>0.7, trade size 5.0
20. **Asset Iteration in Trade Generation**: Iterate through self.assets (sorted list) when generating trade options for deterministic behavior
21. **Results Selection**: Return top 10 paths sorted by final_portfolio_value descending
22. **Value Rounding**: Round flow_cost and final_portfolio_value to 2 decimal places
23. **Edge Cases**: Empty prices -> empty list; time_steps<=0 -> single result with empty trades list, capital_limit value, message "No time steps to simulate."; capital<=0 -> single result with time_steps empty lists, 0.0 value, message "No capital available for trading."; no results→single result with time_steps empty lists, capital_limit value, message "No profitable trading opportunities found."
24. **Price Validation**: Raise ValueError if any initial_asset_price <= 0
25. **Initial State**: Start with all asset positions at 0.0, capital at capital_limit
26. **Final Valuation**: Sum remaining capital plus (position * current_price) for all assets
27. **Memoization Dictionary**: Use standard Python dict for memo storage, keyed by state tuples

"; Always end with "Strategy exploited correlation structure to maximize returns while respecting capital constraints and transaction costs. Final portfolio value: ${final_value:,.2f}."
31. **Edge Case Messages**: Use exactly "No time steps to simulate." for time_steps<=0; "No capital available for trading." for capital<=0; "No profitable trading opportunities found." when simulation produces no results from memoization
