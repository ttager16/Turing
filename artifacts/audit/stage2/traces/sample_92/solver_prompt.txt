Problem Statement:
### Problem Statement

A global consortium of financial regulators seeks a deterministic multilingual analysis system that transforms unstructured financial news into structured, cross-referenced insights. The system must build an evolving article graph, extract key entities from text in multiple scripts, and iteratively propagate sentiment through linked articles while handling new inputs incrementally.

---

### Objectives

1. **Incremental Weighted Graph Construction**

   * Maintain a directed graph where each node is an article and edges encode contextual similarity.
   * Create edges by rule:

     * Entity edge: connect two articles if they share at least two canonical entities; weight = Jaccard(entity sets).
     * Alias edge: connect articles if one contains a lexical alias of an entity in the other; weight = 0 for sentiment transfer (see Sentiment rules).
   * On new article arrival, only update incident edges; existing weights remain unchanged.

2. **Trie-Based Multilingual Entity Recognition**

   * Implement a Trie with canonical forms and aliases. Minimum required canonical buckets:

     * Company: {“Apple Inc.”} with aliases {“Apple Incorporated”, “Apple Inc”}.
     * Stock Symbol: {“AAPL”}.
     * Market Index: {“Nasdaq Composite”, “Nasdaq”} with aliases {“NAZDAQ”, “纳斯达克”}.
   * Fuzzy matching: accept aliases within edit distance ≤ 1; map to the canonical bucket name.

3. **Dynamic Sentiment Propagation**

   * Base sentiment per article from keyword rules (case-insensitive; whole-word match):

     * Positive keywords: {“positive”, “positivos”, “upbeat”, “bullish”, “amplía”, “expands”}.
     * Negative keywords: {“negative”, “negativos”, “bearish”, “cuts”, “recorta”}.
     * If both sets appear, prefer the last sentiment keyword in the article. If none appear, base = neutral.
   * Propagation step: only through Entity edges (alias edges do not transfer sentiment). Convert labels to scores {positive: +1, neutral: 0, negative: −1} and iteratively update each node to the weighted average of neighbors (edge weights normalized). Stop when max absolute change < 0.05 or after 20 iterations. Map back to labels via thresholds: ≥ +0.25 → positive, ≤ −0.25 → negative, else neutral.
   * Deterministic tie-breaking: if score exactly on a threshold, keep prior label.

4. **Safe Concurrency and State Discipline**

   * Ingest concurrently using a thread-safe queue.
   * Node and incident-edge updates must be lock-scoped per node; no global locks.
   * Memory for active graph + trie must stay within implementation limits (no fixed cap required by this prompt).

---

* Average per-article update (entity extraction + local edge updates) is ≤ O(log N) with respect to the number of nodes N, excluding full propagation passes.
* Fuzzy matching must canonicalize at least 90% of single-character spelling variants for entities listed above.
* Sentiment propagation must converge within ≤ 20 iterations using the rule above.
