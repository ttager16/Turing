### Problem Statement

A financial institution seeks to design a long-term investment optimization model that adapts to market fluctuations. The problem is represented as an Infinite Horizon Markov Decision Process (MDP), where each market condition is modeled as a state and each investment choice as an action. The objective is to determine an optimal investment policy that maximizes the expected discounted return across an indefinite future. You must implement the Value Iteration algorithm using Dynamic Programming to find the optimal value function and corresponding optimal policy.

### Objectives

- Implement the Value Iteration algorithm for an infinite horizon MDP.
- Compute the optimal state values representing long-term expected returns.
- Derive the optimal policy mapping each market condition to its best investment decision.
- Continue iterations until the value function converges within a specified tolerance.

**Valid Input Requirements:**

- `transition_probabilities`: nested dictionary where `transition_probabilities[s][a][s_next]` gives the probability of transitioning from state `s` to state `s_next` under action `a`
- `rewards`: nested dictionary where `rewards[s][a]` gives the immediate numeric reward for taking action `a` in state `s`
- `gamma`: float strictly between 0 and 1
- `epsilon`: positive float greater than 0, defining the convergence threshold

**Algorithm Requirements**

- **Initialization:**
  - Set `V(s) = 0.0` for all states s in `states`.

- **Value Iteration Loop:**
  - Repeat the following until convergence:
    1. Store the current value function as `V_old`.
    2. For each state s, compute the new value: `V_new(s) = max_a [ R(s,a) + gamma * Σ_{s'} P(s'|s,a) * V_old(s') ]`
       Where:
       - `max_a` means maximum over all actions in `actions`
       - `R(s,a)` = `rewards[s][a]`
       - `P(s'|s,a)` = `transition_probabilities[s][a][s']`
       - `Σ_{s'}` means sum over all next states s' in `states`
    3. Compute the maximum change:
       `delta = max_s |V_new(s) - V_old(s)|`
    4. If `delta < epsilon`, convergence is achieved; exit the loop.
    5. Otherwise, set `V_old = V_new` and continue to the next iteration.

- **Policy Extraction:**
  - After convergence, for each state s, determine the optimal action:
    `policy(s) = argmax_a [ R(s,a) + gamma * Σ_{s'} P(s'|s,a) * V_final(s') ]`
  - Where `V_final` is the converged value function.

- **Tie-Breaking Rule:**
  - If multiple actions yield the same maximum expected value (within tolerance of 1e-9), select the action that appears first in the `actions` list.

- **Numerical Precision:**
  - Use full floating-point precision for all calculations during iterations.
  - Only round values when producing the final output.

- **Output Formatting:**
  - Round all values in the value function to exactly 2 decimal places using Python's built-in `round()` function.

**Edge Cases**

- **Zero rewards**: If all rewards are zero for all state-action pairs, the value function converges to 0.0 for all states, and the policy selects the first action from the `actions` list for each state (by tie-breaking rule).
- **Single state**: If there is only one state, compute the value iteration normally. The state will have one optimal action.
- **Single action**: If there is only one action, all states will have that action as their optimal policy.
- **Self-loops**: States can transition to themselves with probability 1.0. Handle these transitions normally in the Bellman equation.
- **Deterministic transitions**: When a state-action pair has probability 1.0 for a single next state and 0.0 for all others, compute using the same Bellman equation (the summation will have only one non-zero term).
- **Negative rewards**: Handle negative rewards (losses) correctly in the value iteration, which may result in negative state values.
- **Absorbing states**: States where all actions lead back to the same state with probability 1.0 and reward 0 should converge to value 0.0.
- **Mixed positive and negative rewards**: Handle MDPs where some states have positive rewards and others have negative rewards, ensuring correct value propagation.
- **Sparse transition matrices**: When most transition probabilities are 0.0, the algorithm should still compute correctly for the non-zero transitions.
- **Equal expected values**: When multiple actions for a state have equal expected values (within 1e-9 tolerance), select the action that appears first in the `actions` list.
- **High discount factor**: When gamma is close to 1.0 (e.g., 0.99), convergence may require more iterations but the algorithm terminates when delta < epsilon.
