Problem Statement:
### Problem Statement
Implement a segment tree to process stock market data and compute financial metrics over time intervals. The segment tree must support range sum queries, average calculations, weighted averages, variance computations, and custom aggregation functions over specified ranges. Handle point updates, range updates, and batch operations efficiently using segment tree data structure principles with lazy propagation.

The implementation will process stock prices and their associated weights, allowing queries over arbitrary ranges to calculate financial metrics. The segment tree must store sufficient information to compute sum, average, weighted average, variance, and custom weighted aggregations without recalculating from individual elements for each query. Additionally, support range updates that modify multiple price values simultaneously and handle overlapping query ranges efficiently.


### Objectives:
1. **Range Sum Queries:** Calculate sum of prices over any range [start_idx, end_idx].
2. **Range Average Queries:** Calculate arithmetic mean of prices over any range.
3. **Weighted Average Queries:** Calculate weighted average using formula: sum(prices[i] * weights[i]) / sum(weights[i]) over any range.
4. **Variance Calculations:** Calculate population variance over any range using stored sum and sum-of-squares.
5. **Custom Weighted Aggregation:** Calculate weighted sum with custom weight multipliers: sum(prices[i] * weights[i] * multiplier) over any range.
6. **Point Updates:** Set individual price values to a new value (assignment, not addition) and recalculate affected segment tree nodes.
7. **Range Updates:** Update all prices in a range by adding a constant value using lazy propagation.
8. **Batch Operations:** Process multiple operations (queries and updates) in sequence while maintaining consistency.


### Constraints:
- Receive prices as a list of integers representing stock prices (1 to 10,000) in chronological order.
- Receive weights as a list of floats representing weight factors (0.1 to 10.0) corresponding to each price point.
- Receive multipliers as a list of floats (0.5 to 2.0) for custom weighted aggregation operations.
- Maximum array size: 1,000 price points.
- Maximum number of operations: 500 queries and updates combined.
- All prices, weights, and multipliers will be non-negative values.
- Query ranges are 0-indexed and inclusive of both endpoints.
- Implementation must handle: single-point queries (start_idx == end_idx), multi-point ranges, and full-array queries.
- Cannot use external libraries - implement segment tree from scratch.
- Segment tree must store: sum of prices, sum of weights, sum of (price * weight), sum of (price * price), sum of (price * weight * multiplier), and count for each segment.
- Use this weighted average formula: sum(prices[i] * weights[i]) / sum(weights[i]) for the range.
- Use this variance formula: (sum(prices[i] * prices[i]) / count) - (sum(prices[i]) / count)^2 for the range.
- Use this custom weighted aggregation formula: sum(prices[i] * weights[i] * multipliers[i]) for the range.
- Range updates must use lazy propagation for efficient updates.
- Handle edge cases: single elements and range updates that span multiple segments.
- All floating-point results must be rounded to exactly 2 decimal places.
- Implementation must be consistent: queries after updates must reflect the updated values.

- **Operation format:** Each operation is a list `[name, start_idx, end_idx, value_or_None]` where `name` is one of `"sum"`, `"average"`, `"weighted_avg"`, `"variance"`, `"custom_weighted"`, `"range_update"`, `"point_update"`. For `point_update`, `value_or_None` is the new price value to set at the specified index (note: for point_update, `start_idx` should equal `end_idx`). For queries, `value_or_None` should be `None` (it is ignored by the function).
- **Point update semantics:** `point_update` sets the price at a specific index to a new value (assignment), while `range_update` adds a value to existing prices (addition). - **Out-of-bounds indices:** Ignore portions of a query/update range outside `[0, len(prices) - 1]`. If the entire range lies outside, return `0.0` and perform no update.
- **Zero weights:** If the sum of weights over a requested range is `0.0`, the weighted average is defined as `0.0`.
- **Empty ranges:** For any query whose effective range after intersection has zero elements, return `0.0`.
- **Custom aggregation scope:** "Custom weighted aggregation" refers exclusively to the provided per-index `multipliers` list in the input. Arbitrary user-defined functions are not supported.
