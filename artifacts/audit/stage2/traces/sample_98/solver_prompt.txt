Problem Statement:
### Problem Statement

In a cutting-edge smart agricultural system, a research scientist must optimize both the order of crop placement and the irrigation budget across multiple phases of the growth cycle. The farmland is modeled as a 2D grid where each cell has two attributes: a fertility value (integer) and a water cost (integer) to activate irrigation for planting.

The task is to select a **single connected path** of cells such that fertility values along the path are **strictly increasing** at every step, the **sum of water costs** over all selected cells does **not exceed** the available irrigation budget, and movements are allowed only **up, down, left, or right**. Each cell may appear **at most once** in the path. Cells with equal fertility cannot be adjacent in the path since the increase must be strict. The cost of a path is the sum of water costs of all distinct cells in that path; the starting cell’s cost counts toward the budget.

To enforce determinism and comparability across solutions, a **total order** over cells is defined: cells are first ordered by increasing fertility, then by row index, and then by column index (row-major order). Transitions are permitted **only** from a cell to one of its four neighbors whose fertility is **strictly greater**; transitions between cells with equal fertility are prohibited even if they are neighbors. The planning must be efficient and iterative, suitable for resource-constrained edge deployments.

### Objectives

1. Implement the function to return a **single integer** equal to the **maximum path length** achievable under all rules.

2. Apply **deterministic tie-breaking** among paths that achieve the same **maximum length** as follows, in strict priority order:

   1. Choose the path with the **lowest total water cost**.
   2. If still tied, choose the path whose **starting cell has the lowest fertility**.
   3. If still tied, choose the path whose **starting cell** has the smallest **(row, column)** in row-major order.
   4. If still tied, compare a deterministic **two-modulus rolling hash signature** of the path’s sequence of flattened indices `x = r*cols + c` and choose the lexicographically smaller pair `(hash1, hash2)`.
      If the hash pairs are equal, break ties by the path’s **ending cell (row, column)** in row-major order.
      If both the hash pair and the ending cell are equal, treat candidates as **identical for dominance**.

   **Rolling-hash details (required for determinism):**

   * Maintain two 64-bit hash accumulators with distinct bases. Use **unsigned 64-bit wraparound** (i.e., modulo `2^64`).
   * Fixed constants: `BASE1 = 911382323`, `BASE2 = 972663749`.
   * Initialize `h1 = 0`, `h2 = 0`. For each appended cell id `x = r*cols + c`, update
     `mask = (1 << 64) - 1`
     `h1 = ((h1 * BASE1) + (x + 1)) & mask`
     `h2 = ((h2 * BASE2) + (x + 1)) & mask`
     (The `+1` prevents leading-zero ambiguity.)
   * The comparison in step 2.4 is lex order on `(h1, h2)`.

3. Ensure that the algorithm is **iterative** and **stack-safe**. Solutions that rely on deep recursion or per-start full DFS enumerations are not acceptable. A correct approach may compute DP states in **nondecreasing fertility** order and relax transitions to neighbors with strictly higher fertility, caching the best **(length, cost, start_fert, start_r, start_c, hash1, hash2, last_r, last_c)** for each cell.

4. Guarantee **deterministic output** for identical inputs, independent of hash iteration order or incidental dictionary ordering. Any sorting must be stable with the defined total order. When iterating neighbors, use a fixed order **Right, Left, Down, Up** to avoid incidental nondeterminism.

5. Respect **budget-feasible dominance**. For a given destination cell, a candidate state
   `S = (L, C, start_fert, start_r, start_c, h1, h2, last_r, last_c)` **dominates** another `T` if:

   * `L > L_T`, or
   * `L == L_T` and `C < C_T`, or
   * `L == L_T`, `C == C_T`, and
     `(start_fert, start_r, start_c, h1, h2, last_r, last_c)` is lexicographically smaller than the same tuple of `T`.
     Only **undominated** budget-feasible states may be retained/propagated.

### Constraints
