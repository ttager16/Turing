Problem Statement:
* Context
  A financial technology firm is deploying a real-time portfolio allocator that must keep running during volatile market bursts and occasional data glitches. The engine consumes synchronous arrays of prices and expected returns and allocates a single budget across assets using a deterministic greedy rule. Inputs may contain anomalies such as missing values, non numeric strings, or nonpositive prices. The allocator must sanitize inputs without crashing, record issues for auditability, and still compute a best effort allocation using only in memory operations. No file or network I O is allowed; results must be reproducible across runs and Python versions.

* Objective
  Implement a single Python function that greedily allocates a budget across N assets to maximize a linear utility based on expected_return - risk_penalty per unit currency, subject to per asset maximum weight caps and a total budget. The function must be deterministic, handle bad data via exception safe sanitation, and return a JSON compatible result object with weights, spent, score, and a deterministic list of error messages. Use a single pass greedy allocation over a stable sorted order; do not use dynamic programming or global optimization.

* Constraints

  1. Deterministic ranking: sort assets by descending key k[i] = expected_returns[i] - risk_penalty, then by ascending price[i], then by ascending index i.
  2. Greedy only: allocate in one forward pass over the sorted order without backtracking or reoptimization.
  3. Inputs must be three lists of equal length N for prices, expected_returns, and max_weights; on mismatch, return an empty allocation and include "length mismatch".
  4. If N == 0, return weights=[], spent=0.0, score=0.0, errors=["no assets"].
  5. budget must be a real number strictly greater than 0; otherwise return empty allocation with "invalid budget".
  6. risk_penalty must be a real number; if not, treat as 0 and append "risk_penalty coerced to 0".
  7. prices[i] must be a real number > 0 and finite; otherwise exclude asset i and append a specific error message.
  8. expected_returns[i] must be a real number and finite; if not, exclude asset i and log it.
  9. max_weights[i] must be numeric; values outside 0 to 1 are clipped into 0 to 1 and logged as "max_weights[i] clipped to X".
  10. Excluded assets keep weight 0 but still occupy their index in the output weights list when list lengths matched initially.
  11. Per asset allocation is limited by max_weights[i] * budget.
  12. Allocation uses continuous currency (no share integrality constraints).
  13. After allocation, round each weight, "spent", and "score" to round_decimals places using Python round for reproducibility.
  14. round_decimals must be an int in 0 to 12; if invalid, default to 6 and append "round_decimals coerced to 6".
  15. Sum of allocations must not exceed budget by more than 1e-9 before rounding; after rounding, ensure spent <= budget with a corrective downward adjustment of the most recently changed asset if needed.
  16. Never produce a negative remaining budget; guard against rounding creating negative leftovers by adjusting the last allocated asset downward.
  17. Score equals sum over i of (rounded_weight_i * budget / price[i]) * (expected_returns[i] - risk_penalty). Compute using the rounded weights, then round the final score to round_decimals.
  18. Negative keys are allowed; such assets simply rank lower and may still receive allocation if budget remains and caps permit.
  19. Sorting keys are computed once after sanitation and reused unchanged during allocation.
  20. Stable error ordering: accumulate per index errors in ascending index order; append any global messages afterward. If an asset is excluded due to nonnumeric or nonpositive fields, still log any max_weights[i] clipping for that index.
  21. Input immutability: do not mutate input lists; copy or read only.
  22. No randomness, no time based logic, and no reliance on dict iteration order affecting results.
  23. Complexity target O(N log N) time due to sorting and O(N) extra memory; no recursion.
  24. All outputs must be finite numbers after rounding; convert negative zeros to 0.0 by adding 0.0.
  25. Include "all caps zero" in errors whenever all sanitized max_weights are exactly 0 and no global early exit from constraints 3, 4, 5, or 28 was triggered.
  26. If every asset is excluded during sanitation, return weights of length N with all zeros, spent=0.0, score=0.0, and the collected errors.
  27. Do not perform I O of any kind; no prints, logging, files, or network.
  28. Abort and return empty allocation with "numerical failure" if any intermediate becomes NaN or infinite during computation.
  29. Tie handling is strictly by (key desc, price asc, index asc); do not introduce secondary floating comparisons beyond this rule.
  30. The function must always return a dict with exactly the keys "weights", "spent", "score", "errors".
  31. The function may assume N up to 100000 and must avoid quadratic behavior.
  32. Do not use Decimal, Fraction, numpy, or third party libraries.
  33. Ensure that spent equals sum(weights[i] * budget) within rounding; recompute "spent" from the rounded weights to maintain consistency.
