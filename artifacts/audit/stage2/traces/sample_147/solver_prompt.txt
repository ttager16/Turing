Problem Statement:
## **Problem Statement: Optimizing Multilevel Distribution Flows with State-Dependent Constraints**

In a modern logistics network, multiple distribution centers and transport routes are interconnected to deliver shipments efficiently under time and capacity constraints. Each possible route between two centers is represented as a **directed edge** in a graph.
Every edge has three properties:

* **Travel Time:** the time taken to move goods through that route.
* **Route Capacity:** the maximum capacity or load that can be transported on that route.
* **Transit Cost:** the cost incurred to transport goods through that route.

The task is to design a Python program that finds a single optimal delivery path that passes through all mandatory checkpoints in order, minimizing total transit cost while ensuring that cumulative travel time and capacity constraints are not violated.

To solve this, the algorithm should dynamically track **state information** (like partial loads and remaining time) at each node and update it as shipments progress through routes. The goal is to identify **feasible flow paths** that satisfy all constraints while minimizing total cost.

Although this problem borrows terms from logistics flow modeling, it should be interpreted as a constrained path optimization problem, not a full network flow optimization. The goal is to determine one feasible route that minimizes total cost under time and capacity limits.

---

## **Objectives**

1. **Routing Graph Construction**

   * Represent the logistics network as a **directed graph** where each node denotes a distribution center or a transfer junction.
   * Each edge is represented as a list containing
     `[source_node, destination_node, travel_time, route_capacity, transit_cost]`.

2. **Path-State Tracking**
Maintain a path state at each node to track:
• Cumulative travel time so far.
• Cumulative capacity usage so far.
Update these state variables as the route expands, ensuring that no constraint is exceeded during the search.

3. **Constraint Enforcement**

   * **Global Time Constraint:** The sum of travel times along any route must not exceed `global_max_time`.
   * **Global Capacity Constraint:** The total route capacity used simultaneously must not exceed `global_max_capacity`.
   * **Mandatory Checkpoints:** Specific nodes in `mandatory_checkpoints` must appear in all routes in the same given order.
   * **Per-Edge Costs:** Each route has an associated cost; the algorithm should minimize the total cost of all selected routes.

---

## **Constraints**

* `1 ≤ num_nodes ≤ 100`
* `1 ≤ len(edges) ≤ 1000`
* Each `travel_time`, `route_capacity`, and `transit_cost` is a positive integer (≤ 10⁶).
* Each edge is a **directed connection** `[u, v, travel_time, route_capacity, transit_cost]`.
* The total time of any path ≤ `global_max_time`.
* The total load in transit ≤ `global_max_capacity`.  
  (For simplicity, this refers to the **sum of route capacities along a single selected path**, not across multiple concurrent routes.)
* All checkpoints in `mandatory_checkpoints` must be visited **in the given order**.
* Nodes may repeat in routes if the shortest feasible path revisits a distribution center.

---

## **Key Definitions**

 **Node**                  
 A distribution center or key transfer point in the logistics network.              
 **Edge**                  
 A directed route between two nodes with defined time, capacity, and cost.               
 **State**                 
The cumulative record of total travel time, total capacity used, and total cost accumulated along a candidate path so far.
 **Mandatory Checkpoint**  
 A node that must appear in a valid delivery route sequence in the specified order. 
 **Global Constraints**    
 Overall time and capacity limits applied to the entire transportation process.
