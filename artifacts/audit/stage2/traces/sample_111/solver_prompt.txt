# Interactive Max-Flow Simulator Prompt

## Real-World Narrative:
The University Mathematics Department is creating an interactive Python-based tool to teach graduate students the fundamentals of network flow through the max-flow problem. The tool aims to provide an engaging platform for visualizing dynamically generated directed graphs, enabling students to adjust edge capacities and observe real-time updates to flow distributions. By integrating a step-by-step exploration of the Ford-Fulkerson algorithm, the tool will help students understand how augmenting paths are selected and how maximum flow is achieved. Detailed explanations accompanying each step will clarify the algorithm’s mechanics, flow conservation, and capacity constraints, fostering a deep understanding of network flow concepts in an educational setting.

## Objectives:
1. **Max-Flow Computation**  
   Implement a module to compute the maximum flow in a directed graph from a source to a sink, ensuring accurate handling of edge capacities and flow conservation.

2. **Dynamic Graph Manipulation**  
   Support operations to add, remove, or modify edge capacities in real-time, with immediate updates to the flow distribution and visualization to reflect changes.

3. **Algorithm Step-Through**  
   Provide an interactive interface for stepping through the Ford-Fulkerson algorithm, allowing students to pause, rewind, or fast-forward through augmenting path selections and flow updates, with visual feedback on residual graphs.

4. **Flow Visualization and Analytics**  
   Display real-time flow values per edge, total maximum flow, and residual capacities. Highlight augmenting paths and bottleneck capacities during each iteration.

5. **Detailed Explanations**  
   Include concise explanations (<200 words each) for each algorithm step, addressing flow updates, capacity constraints, and edge cases (e.g., zero-capacity edges, unreachable sinks, or multiple augmenting paths).

## Constraints:
- The tool must be written in Python without external libraries for graph or visualization (e.g., use Tkinter or text-based output).  
- It must handle graphs with up to 100 nodes and 500 edges efficiently.  
- The implementation should use the Edmonds-Karp algorithm (BFS-based Ford-Fulkerson) with O(V * E²) time complexity, where V is the number of vertices and E is the number of edges.  
- Explanations must be clear, concise (<200 words), and educational, covering key concepts like bottleneck capacity and flow conservation.  

## Advanced Edge Case Constraints:
1. **Anti-Parallel Edges (Bidirectional Flow)**  
   The algorithm must correctly handle graphs with anti-parallel edges (edges going in both directions between the same pair of nodes). The implementation should maintain separate flow values for each direction and properly update the residual graph to avoid conflicts. Example: If edges (u→v, capacity 10) and (v→u, capacity 5) both exist, the algorithm must treat them independently and ensure flow conservation holds for both directions.

2. **Backward Edge Utilization in Residual Graphs**  
   The implementation must demonstrate correct handling of scenarios where optimal flow requires extensive use of backward edges in the residual graph. This occurs when initially selected augmenting paths are suboptimal and the algorithm must "undo" previous flow decisions by routing flow backward to find better paths. Test cases should include graphs where the naive greedy approach fails without proper residual graph management.

3. **Tie-Breaking with Multiple Equal-Capacity Paths**  
   The algorithm must handle graphs with exponentially many augmenting paths of equal capacity without degradation in performance or correctness. When multiple paths have identical bottleneck capacities, the BFS-based path selection should be deterministic and consistent. The implementation should efficiently resolve ties and demonstrate that the final maximum flow value is independent of the specific path selection order, while the number of iterations may vary.
