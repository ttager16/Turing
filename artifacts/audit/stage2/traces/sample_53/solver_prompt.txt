### Problem Statement

In a multi-exchange high-frequency trading (HFT) ecosystem, traders must simultaneously track numerous assets with non-linear price dependencies, evolving correlation graphs, and regulatory constraints. This problem involves constructing a dynamic model that captures arbitrage potentials and transaction costs across multiple interconnected trading venues. The goal is to compute arbitrage differentials between assets after all price updates are applied.

At its core, you must design a graph-based arbitrage detection function that, after applying updates, evaluates the cost difference (arbitrage differential) between asset pairs. Each edge in the graph represents a possible trade path between two assets with an associated transaction fee or rebate. A rebate is represented as a negative edge weight.


### Objectives

1. Build a robust graph structure to represent trading venues, assets, and edges with transaction costs or rebates.
2. Apply all price updates in order before processing any query.
3. For each query, compute the arbitrage differential using the direct-edge preference rule. If no direct edge exists, compute the minimum summed fee path or detect if a negative cycle affects the route.
4. Return the results for all queries in a single batch evaluation.

### Constraints

- Graph size: up to 10^4 nodes and 5×10^4 directed edges.
- Apply all price updates before answering queries.
- Edge weights are absolute fees or rebates (same currency as prices); negatives allowed.

- The function returns a list of floats in the order of the input queries.
