## Problem Statement
A municipal water utility company operates a complex urban water distribution network where maintaining optimal flow rates is critical for ensuring reliable service while preventing reservoir overflow and pipe damage. The network is modeled as a directed graph where water treatment plants serve as sources, reservoirs act as intermediate storage nodes, and residential/commercial areas represent demand sinks.

You are tasked with implementing a sophisticated Flow Network Optimization algorithm that manages water distribution through this multi-tiered network. The system must efficiently detect flow cycles, identify bottlenecks, and optimize water routing while respecting capacity constraints, pressure requirements, and demand fulfillment across multiple priority zones.

## Optimization Approach
The algorithm implements **minimum-flow optimization** - it finds the optimal flow allocation that satisfies all demand requirements using the minimum total network flow. This approach minimizes operational costs, reduces system stress, and maximizes efficiency by avoiding unnecessary flow through pipes. The algorithm uses Ford-Fulkerson max-flow principles to find augmenting paths from sources to demand zones, allocating only the flow necessary to meet each zone's target requirements in priority order.

## Core Requirements
Your solution must implement a **pure Python** Flow Network Optimization algorithm (no external libraries) that processes urban water distribution data and generates optimal flow allocation plans. The algorithm should satisfy all demand zone requirements while minimizing total network flow usage and ensuring system stability.

### Key Operational Constraints
1. **Flow Conservation**: Total inflow must equal total outflow at every intermediate node (excluding sources and sinks)
2. **Capacity Constraints**: Flow through each pipe segment cannot exceed its maximum capacity
3. **Pressure Requirements**: Each demand zone must receive a minimum flow to maintain adequate pressure
4. **Priority Allocation**: Higher priority zones (lower priority numbers) must be satisfied before lower priority zones
5. **Cycle Detection**: The algorithm must identify and handle flow cycles that could cause system instability

### Advanced Algorithmic Requirements
The Flow Optimization algorithm must incorporate a multi-objective scoring system that evaluates network efficiency using this composite formula:

**Network Efficiency Score = (Total Flow Delivered / Total Available Capacity) × 100 + Priority Satisfaction Bonus - Cycle Penalty**

Where:
- Total Flow Delivered: Sum of actual flow reaching all demand zones
- Total Available Capacity: Sum of the maximum capacities of all outgoing edges from sources
- Priority Satisfaction Bonus: Sum of (flow_delivered × (6 - priority_level) × 10) for each satisfied zone
- Cycle Penalty: Number of detected cycles × 50

The algorithm must also implement pressure drop calculation, where pipes with high utilization rates experience reduced effective capacity:

**Pressure Drop Formula**:
- If flow_rate / pipe_capacity > 0.8: effective_capacity = pipe_capacity × 0.9
- If flow_rate / pipe_capacity > 0.9: effective_capacity = pipe_capacity × 0.8
- Otherwise: effective_capacity = pipe_capacity

**Flow Redistribution Rule**:
When cycles are detected, the algorithm must redistribute flow by reducing the minimum flow in the cycle by 20% and reallocating it to alternative paths using the following criteria:
- Select pipes not part of the detected cycle
- Choose pipes with `maintenance_status` = "active" 
- Prioritize pipes with current utilization < 80% (current_flow_rate < max_flow_capacity × 0.8)
- Distribute redistributed flow evenly among qualifying alternative pipes
- Respect each pipe's remaining capacity (max_flow_capacity - current_flow_rate)

**Additional Calculation Definitions**:
- **satisfaction_percentage**: (flow_delivered / target_flow_requirement) × 100
- **pressure_status**: "normal" if utilization ≤ 80%, "reduced" if utilization ≤ 90%, "critical" if utilization > 90%
- **total_production_cost**: Sum of (current_production_rate × operational_cost) for all sources
- **utilization_rate** (reservoirs): current_water_level / max_storage_capacity
- **underutilized_capacity**: Sum of (max_flow_capacity - current_flow_rate) for all pipes with utilization < 50%
- **bottleneck_pipes**: List of pipe_ids where utilization_percentage > 95%

## Implementation Guidelines
**Flow Network Strategy**: Implement Ford-Fulkerson max-flow algorithm with breadth-first search to find augmenting paths from sources to demand zones
**Priority Processing**: Process demand zones in priority order (priority_level 1 first, then 2, etc.)
**Minimum Flow Optimization**: Allocate only the minimum flow necessary to satisfy each zone's target requirements
**Capacity Management**: Apply pressure drop calculations dynamically based on allocated flows
**Cycle Resolution**: When cycles are detected, redistribute flow using the 20% reduction rule and find alternative paths
**Reservoir Monitoring**: Track water levels and prevent overflow by redirecting excess flow to alternative reservoirs
**Cost Optimization**: Use minimum flow allocation to minimize operational costs
**Bottleneck Identification**: Identify pipes operating at >95% capacity as bottlenecks requiring attention
**Deterministic Output**: The algorithm must produce consistent results for identical inputs
**Maintenance Integration**: Reduce effective capacity by 30% for pipes in "maintenance" status and 50% for "degraded" status

## Algorithm Flow
1. **Preprocessing Phase**: Validate inputs and build network graph representation using current_flow_rate for capacity validation
2. **Priority Sorting**: Sort demand_zones by priority_level in ascending order
3. **Flow Network Construction**: Build capacity matrix representing available pipe capacities, accounting for current_flow_rate utilization
4. **Cycle Detection**: Use depth-first search with current_flow_rate values to identify flow cycles and mark problematic paths
5. **Flow Redistribution**: If cycles exist, apply 20% reduction to current_flow_rate and redistribute to alternative paths
6. **Flow Allocation**: Apply the Ford-Fulkerson algorithm starting from the updated flow state to calculate new allocated_flow_rate for each zone in priority order
7. **Pressure Analysis**: Calculate pressure status for each pipe segment based on final allocated_flow_rate utilization rates
8. **Reservoir Management**: Monitor reservoir levels using net flow changes from allocated_flow_rate and implement overflow prevention measures
9. **System Optimization**: Minimize total flow usage while satisfying all demand requirements using allocated_flow_rate as final output
10. **Result Compilation**: Generate comprehensive output with allocated_flow_rate as optimized values replacing initial current_flow_rate

The algorithm should terminate successfully when all feasible demand zones are served within their requirements using minimum total network flow, or identify zones that cannot be satisfied due to network capacity limitations or connectivity issues.
