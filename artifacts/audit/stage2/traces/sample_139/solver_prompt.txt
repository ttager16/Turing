### Problem Statement
Urban transportation networks require sophisticated routing algorithms that can adapt to dynamic traffic conditions and infrastructure disruptions. The algorithm must handle complex edge-based robustness constraints where roads have different reliability levels and can fail independently, requiring alternative routing strategies that maintain connectivity even under adverse conditions.

The system must process a city graph where each edge represents a road segment with varying congestion multipliers, reliability scores, and peak hour adjustments. The algorithm should compute paths that not only minimize travel time but also maximize robustness against potential road closures and traffic spikes. This requires implementing advanced shortest path algorithms with edge-based robustness constraints, enabling city planners to enhance traffic flow and improve urban mobility resilience.


### Objectives:
1. Find the shortest path between two nodes while considering edge reliability and congestion factors.
2. Apply congestion multipliers to edge weights based on current traffic conditions.
3. Ensure selected paths maintain minimum reliability thresholds to handle potential road closures.
4. Adjust path weights during peak hours to account for increased travel times and congestion.
5. Select paths that meet minimum reliability requirements.
6. Select the path with minimum travel time among all paths meeting reliability requirements.

- base_weight: Integer representing base travel time (1 to 1000 minutes).
- congestion_multiplier: Float representing current congestion level (1.0 to 5.0, where 1.0 is no congestion).
- reliability_score: Float representing road reliability (0.1 to 1.0, where 1.0 is fully reliable).
- peak_hour_adjustment: Float representing additional time during peak hours (1.0 to 3.0 multiplier).
- start_node: String representing the starting node ID ("0" to "n-1").
- end_node: String representing the destination node ID ("0" to "n-1").
- is_peak_hour: Boolean indicating if current time is peak hour.
- min_reliability: Float representing minimum acceptable reliability threshold (0.1 to 1.0).
- robustness_weight: Float representing weight given to robustness vs travel time (0.1 to 2.0).
- Maximum number of nodes: 1000.
- Maximum number of edges per node: 50.
- All edge weights must be positive integers.
- Path must exist between start and end nodes.
- Algorithm must use Dijkstra's algorithm with robustness modifications.
- Cannot use external graph libraries - implement from scratch.
- Handle edge cases: single node graphs (return [start_node] if start_node == end_node) and disconnected components (return empty list).
- Robustness score calculation: sum of (reliability_score * robustness_weight) for all edges in path.
- Final path weight calculation: if is_peak_hour is True, sum of (base_weight * congestion_multiplier * peak_hour_adjustment) for all edges in path; if False, sum of (base_weight * congestion_multiplier) for all edges in path.
- Path selection criteria: among all paths with robustness_score >= min_reliability, select the one with minimum final_weight.
- If no path meets reliability threshold, return empty list.
