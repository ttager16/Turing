### Problem Statement

A metropolitan logistics company operates a multi-layer delivery network with three transportation layers: ground roads, elevated skyways, and underground tunnels. Each layer has distinct characteristics with different speeds and access points. Your task is to implement a backtracking-based pathfinding algorithm that determines optimal delivery routes for multiple vehicles while respecting layer-specific constraints, time windows, vehicle capacity limits, and dynamic route closures. The algorithm must use heuristic pruning to efficiently explore the solution space and process multiple vehicles independently.

---

#### Objectives:

1. Multi-Layer Route Planning: Implement backtracking algorithm to explore routes across three transportation layers (roads, skyways, tunnels) where vehicles can transition between layers at designated junction nodes with transition costs.

2. Heuristic-Guided Pruning: Apply heuristics to prune search branches early including time window violations, capacity feasibility checks, and cost bounds. Use admissible heuristics (calculate the true shortest path cost between any two nodes using Dijkstra's algorithm on the weighted graph, never overestimating) to guide path exploration and prune branches that are guaranteed to exceed the best-known solution.

3. Dynamic Route Closure Handling: Handle route closures that block specific edges during planning. When a closure affects the current path, backtrack and explore alternative routes through different layers without complete recalculation.

4. Independent Multi-Vehicle Routing: Route multiple delivery vehicles independently. Each vehicle is processed separately without shared resource constraints.

5. Delivery Constraints: Ensure all required delivery nodes are visited before reaching the end node, respecting vehicle capacity limits (number of deliveries cannot exceed capacity).

---

#### Constraints:

- City map contains up to 100 nodes distributed across 3 layers with up to 300 edges total
- Each edge has cost (integer time units) and layer designation (road, skyway, or tunnel)
- Vehicles (up to 10) have capacity limits (1 to 100 units), time windows (start time, end time), and assigned delivery nodes
- Layer transition nodes follow naming convention: base nodes like "N1", layer variants like "N1_skyway", "N1_tunnel"
- Transitioning between layers at junction nodes adds fixed cost of 5 time units per transition
- Route closures provided as (from_node, to_node, closure_time) tuples - edge unavailable at or after that time
- Algorithm must use backtracking with pruning - no external pathfinding libraries allowed
- Return empty list [] only if no vehicles can be routed successfully (all vehicles fail constraints)
- Primary objective: minimize total delivery time. Secondary objective: minimize layer transitions
- Solution must work with Python standard library only
