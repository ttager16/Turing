### Problem Statement

A financial analytics firm is developing a Python-based system to analyze transaction records from multiple global markets. These records exhibit high variance in transaction amounts due to currency fluctuations and market differences. The firm requires a custom sorting algorithm that efficiently handles this variance, ensuring rapid organization of transaction data for real-time analysis and reporting. The solution must be scalable, capable of processing large datasets with unequal key distributions, and adaptable to dynamically changing market conditions, to provide timely insights for decision-making.

The module accepts a dictionary mapping market identifiers to their transaction amount arrays (floats already normalized to a reference currency such as USD). It must return a new dictionary where each market’s transactions are sorted in non-decreasing numeric order and a final combined list that merges all markets in a globally sorted, deterministic, and stable manner. The module must strictly validate input schema and data types, preserve full floating-point precision, avoid mutating the input, and fail fast on any violation.


### Objectives

1. Deterministic and stable numeric sorting
   * Sort each market’s list of floats in non-decreasing numeric order.
   * Preserve duplicate values and maintain stability for equal values.
2. Multi-market stable merge
   * Produce a single combined list by a deterministic, stable, multi-round pairwise merge of all individually sorted market lists.
   * Merge schedule must be deterministic and based on lexicographic order of market identifiers.
3. Performance and scalability
   * Validation O(n) over all values.
   * Per-market sorting O(n_i log n_i).
   * Global merge O(N log T) for N total values across T markets.

- The solution must be implemented in pure Python (no external libraries allowed).

  * Implement iterative bottom-up merge sort for each market list.
- Input structure (any violation must return {})
  * The input must be a non-empty dictionary where each key consists of english alphabets and non-empty strings.
  * Each value must be a list.
  * Every element of each list must be exactly a float and finite.
  * Empty lists are permitted for markets that currently have no transactions.
- Sorting and ordering rules
  * Sorting must be numeric and stable in non-decreasing order.
  * After sorting all markets, build combined by a deterministic multi-round pairwise merge:

    * Merge adjacent lists pairwise to form the next round, repeat until one list remains.
    * Each merge must be linear-time and stable and must not rely on heaps, priority queues, or re-sorting.
  * The returned dictionary must contain all market keys in lexicographic order, plus the final key combined as the last key.
