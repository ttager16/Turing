Problem Statement:
### Problem Statement
In a cybersecurity firm, a team of engineers is tasked with analyzing a network of interconnected systems to detect potential vulnerabilities. The network consists of interconnected systems where each system has a vulnerability score, trust level, and system type, and connections represent communication channels with different security levels.

The engineers must implement both recursive and iterative versions of Depth-First Search (DFS) to analyze the network topology and identify critical attack paths. The analysis requires finding paths from a starting system to target systems, calculating cumulative vulnerability scores along each path, and determining critical paths that attackers could exploit.

### Objectives:
1. Implement a recursive DFS that traverses the network and identifies paths from source to target systems.
2. Implement an iterative DFS using explicit stack management.
3. Calculate cumulative vulnerability scores for each discovered path using threat-model-specific formulas.
4. Identify paths with vulnerability scores exceeding threshold values.
5. Handle cycles in the network graph during traversal.
6. Return only paths that meet vulnerability threshold and length constraints.
7. Implement system type compatibility checking to prevent invalid attack paths.
8. Support multiple threat models with different vulnerability calculation methods.
9. Validate privilege escalation requirements along attack paths.

### Constraints:
- Connection security levels: Integers from 1 (most secure) to 5 (least secure).
- Trust levels: Integers from 1 (lowest trust) to 10 (highest trust).
- System type compatibility rules: server(1) connects to all types, workstation(2) connects only to server(1) and database(3), database(3) connects only to server(1) and workstation(2), network_device(4) connects to all types, security_appliance(5) connects only to server(1) and network_device(4).
- Privilege escalation rules: user(1) can escalate to admin(2), admin(2) can escalate to root(3), root(3) can escalate to system(4). Same privilege level transitions are allowed. Higher privilege levels can access lower privilege levels.
- Cannot use external graph libraries - implement DFS from scratch.
- Cannot use tuples or tuple data structures
- Both implementations must return identical results for the same input.
- Handle disconnected components - return empty paths for unreachable targets.
- Must detect and handle cycles - paths must not contain repeated system IDs.
- Paths must be returned as lists of system IDs in traversal order.
- Only return paths that meet the vulnerability threshold requirement.
- Paths must respect system type compatibility constraints.
- Paths must satisfy privilege escalation requirements (each step must be valid according to privilege escalation rules).
- Output dictionary keys must be strings.
- Malformed edges (lists with fewer than 3 elements) must be skipped without raising errors; they do not contribute to traversal or scoring.

### Input Specifications:
- network: Dictionary where keys are system ID strings "0" to "9999" and values are lists of lists [connected_system_id, connection_security_level, trust_level].
- vulnerability_scores: Dictionary mapping system ID strings to vulnerability scores (0.0 to 10.0).
- system_types: Dictionary mapping system ID strings to system types (1=server, 2=workstation, 3=database, 4=network_device, 5=security_appliance).
- privilege_levels: Dictionary mapping system ID strings to privilege levels (1=user, 2=admin, 3=root, 4=system).
- source_system: Integer representing the starting system ID for DFS traversal.
- target_systems: List of integers representing target system IDs to find paths to.
- vulnerability_threshold: Non-negative float (0.0+) representing minimum vulnerability score for critical paths. No upper bound.
- max_path_length: Integer (1 to 50) representing maximum allowed path length.
- threat_model: String indicating threat type ("insider", "external", "apt") affecting vulnerability calculation.

### Vulnerability Calculation Formulas:
- Insider threat: sum(vulnerability_scores[v] * connection_security_level / trust_level * 1.5) for each edge
- External threat: sum(vulnerability_scores[v] * connection_security_level / trust_level * 0.8) for each edge  
- APT threat: sum(vulnerability_scores[v] * connection_security_level / trust_level * 2.0) for each edge
Note: Cumulative vulnerability scores are not normalized or clamped; they can exceed 10.0 depending on path length and factors. Choose thresholds accordingly.
