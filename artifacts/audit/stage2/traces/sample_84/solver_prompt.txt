Problem Statement:
### Problem Statement
Design an efficient engine to process a sequence of range operations on an integer array with reversible updates, richer query modes, and auditable output.

The engine must support these operation types (each operation is a list):
1. Update Operation: add `delta` to all elements in range [L, R] (inclusive)
2. Query Operation: return aggregate over [L, R] by mode: `sum`, `min`, `max`, `mean`, `var`, `unique`, `rms`, `trend`
3. Reverse Operation: undo a previous update operation by its positive integer ID
4. Batch Operation: apply multiple range updates atomically as a group
5. Conditional Query: return range sum only if it exceeds a threshold, else return 0

### Objectives
1. Handle up to 10,000 array elements and 10,000 operations
2. Support operation rollback by tracking update history and preventing double-reverse
3. Provide deterministic results, a per-operation audit trail, and a run-level summary

### Function Signature
```python
def process_stock_movements(n: int, operations: list) -> dict:
```

### Constraints
- 1 ≤ n ≤ 10,000 (array size)
- 1 ≤ q ≤ 10,000 (number of operations)
- −1,000,000 ≤ value (initial or delta) ≤ 1,000,000
- Array indices are 1-based and inclusive
- Operation types (each op is a list):
  - `['update', L, R, delta]`
  - `['query', mode, L, R]` where `mode` ∈ {'sum','min','max','mean','var','unique','rms','trend'}
  - `['reverse', operation_id]`
  - `['batch_update', [[L1, R1, d1], [L2, R2, d2], ...]]` (k ≤ 100)
  - `['conditional', L, R, threshold]` (sum if sum > threshold else 0)
- Each atomic update (including inside a batch) receives a sequential positive ID starting at 1
- Batch size: k ≤ 100 sub-operations
- For all operations, cumulative values and aggregates must remain within [−2^63, 2^63−1]
- No external libraries allowed (standard library only)

### Input Specifications
- `n`: Positive integer (length of array, initially all zeros)
- `operations`: List of list-encoded operations as specified above

### Output Format
- On success, return a dictionary with keys:
  - `results`: List of integer answers for each query-type op in the order encountered. Rounding/typing rules below.
  - `audit`: List of dictionaries, one per processed operation, with keys:
    - `op_idx`: int (1-based index of the processed operation)
    - `type`: string (e.g., 'update','query','reverse','batch_update','conditional')
    - `args`: list (the operation arguments as lists/strings, JSON-compatible)
    - `result`: int or null (query → integer result; non-query → null)
    - `affected_range`: list or list of lists ([L,R] or [[L1,R1],...])
  - `summary`: Dictionary of run-level statistics:
    - `num_updates`: int (including sub-ops in batches)
    - `num_queries`: int (includes 'query' and 'conditional')
    - `num_reverses`: int
    - `num_batches`: int
    - `final_sum`: int (sum of entire array at end)
    - `final_min`: int
    - `final_max`: int
    - `final_unique`: int (number of distinct values in final array)

### Query Modes and Rounding
* `sum`, `min`, `max`, and `unique`: return integers
* `mean`: rounded to the nearest integer using HALF_UP rounding (i.e., values ending in .5 are rounded away from zero)
* `var`: population variance over [L,R], rounded to the nearest integer using HALF_UP rounding
* `rms`: root-mean-square over [L,R], rounded to the nearest integer using HALF_UP rounding
* `trend`: least-squares slope of values vs. indices in [L,R], scaled by 1000 and rounded to the nearest integer using HALF_UP rounding (milli-units). Indices use 1-based positions.

### Error Handling
Return a dictionary with key `error` and one of the following exact strings (do not raise exceptions). The first error matching any input or runtime condition MUST be returned immediately without further processing.

- "n must be a positive integer": if n is missing, not an integer, or not > 0
- "operations must be a list": if operations argument is not a list
- "operation at index {i} must be a list": if any operation in operations is not a list
- "operation at index {i} has invalid type or argument count": if the operation type string or argument count does not match any allowed operation form

- "delta at op {i} must be an integer": if delta is missing or not an integer in update/batch_update
- "batch_update at op {i} exceeds max batch size": if batch_update includes more than 100 sub-operations
- "operation type not supported at op {i}": if operation type string is not one of the allowed operation types
- "conditional query at op {i} requires integer threshold": if conditional query threshold is missing or not integer
- "query mode not supported at op {i}": if mode is not among allowed query modes
- "cumulative value out of range after op {i}": if any array value/result exceeds int64 limits ([−2^63, 2^63−1]) after any operation
- "reverse references non-existent or already reversed op id {id}": if reverse targets non-existent or already reversed op id
- "operation id must be a positive integer": if referenced operation id is missing or not a positive integer
- "unknown error at op {i}": any internal failure not matching previous error cases
