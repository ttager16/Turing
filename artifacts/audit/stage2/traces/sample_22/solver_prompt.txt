Problem Statement:
### Problem Statement

Simulate a sequence of **classical** quantum-style operations on up to 32 qubits encoded in a single integer bitmask (LSB = qubit 0). This simulator operates on a **single** computational-basis state (no amplitude vectors). It supports multi-controlled bit flips (MCX / multi-controlled X), multi-target CNOTs, deterministic measurements (read a bit), reordering (permutation) of qubit indices, and optional state tracing for debugging and analysis.
The simulator processes a provided sequence of gates in order and returns the resulting state.

---

#### Objectives

1. Implement correct semantics for supported operations.
2. Support `MCX` (controls then targets), `CNOT`, `MEASURE`, and `REORDER` (full permutation list) operations. `H` and arbitrary rotations are **not** supported in this mode (they create superposition).
3. Provide stable error handling: invalid qubit indices or invalid permutations should **return a string error message** (not raise a Python exception). The function must return a descriptive string (e.g., `"ValueError: Invalid Input"`) and leave state unchanged.
4. **Track changed qubits:** The simulator should identify which qubits were modified by the sequence of gates and include this information in the output (e.g., `"Changed qubits: [2,3,5]"`).
5. **Support optional state tracing:** When a `"TRACE"` gate is included in the instruction list, the simulator must record the current computational-basis state and include a `"Trace:"` section in the final output.

---

#### Constraints

* Up to 32 qubits in a single `int`. If `num_qubits > 32`, the simulator should accept Python `int` (arbitrary precision) and operate identically.

* No external libraries.

* Inputs: `bitmask: str`, `gates: List[List]`, `num_qubits: str` (all numeric values as strings for JSON compatibility).

  * `bitmask` should be a string representing a non-negative integer in binary (`"0b..."`), decimal (`"51"`), or hex (`"0x..."`) format.
  * `num_qubits` should be a string representing an integer ≥ 1 (e.g., `"6"`).
  * Gates: same list formats as before:
    - `['MCX', [c0, c1, ..., t]]`
    - `['CNOT', [control, target]]`
    - `['MEASURE', [q]]`
    - `['REORDER', [new_order_of_indices]]`
    - `['TRACE', []]`

* Single entry point:

  ```python

      ...
  ```

* **Input validation (applies before any computation):**

  * `bitmask` must be a string representing a non-negative integer less than `2 ** num_qubits`.
  * `num_qubits` must be a string representing an integer ≥ 1.
  * Each gate must have valid structure, operation type, and qubit indices in `[0, num_qubits - 1]`.
  * For `"REORDER"`, the provided list must be a full permutation of `[0, 1, ..., num_qubits - 1]`.
  * For `"TRACE"`, the qubit list must be **empty** (`[]`).
  * On invalid input, immediately **return a string error message** (not raise an exception) and leave state unchanged.
  * The error message must be exactly `"ValueError: Invalid Input"` (as a string) for any invalid input.

* **Time complexity constraint:**
  The function should run in **O(G * Q)** or better, where `G = number of gates` and `Q = number of qubits`.
