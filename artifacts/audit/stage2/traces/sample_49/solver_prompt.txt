## Problem Statement

A multinational retail corporation operates a complex supply chain network with multiple distribution centers, warehouses, and delivery destinations. The company needs to optimize transportation routes to minimize total operational costs while satisfying strict delivery constraints and capacity limitations.

You are tasked with implementing a sophisticated greedy algorithm that dynamically selects optimal transportation routes based on real-time operational data. The system must handle multiple distribution centers, each with different capacities, operating costs, and delivery time windows, while ensuring all customer demands are met within specified deadlines.

## Core Requirements

Your solution must implement a **pure Python** greedy algorithm (no external libraries) that processes supply chain data and generates an optimal route assignment plan. The algorithm should prioritize routes based on cost-efficiency while respecting all operational constraints.

### Key Operational Constraints

1. **Capacity Constraints**: Each distribution center has a maximum daily throughput capacity (sum of all assigned demands cannot exceed capacity)
2. **Time Window Constraints**: Each delivery must be scheduled within the specified time window for that destination
3. **Demand Fulfillment**: All customer demands must be satisfied exactly
4. **Route Exclusivity**: Each destination can only be assigned to one distribution center
5. **Cost Optimization**: The algorithm must minimize total operational costs

### Advanced Algorithmic Requirements

The greedy selection criterion must consider multiple factors simultaneously using this composite scoring function:

**Total Route Cost = Transportation Cost + (Base Cost × Demand × Efficiency Factor)**

Where:
- Transportation Cost: From cost_matrix[distribution_center][destination]
- Base Cost: Per-unit operational cost from distribution center
- Demand: Delivery quantity for the destination
- Efficiency Factor: Processing efficiency multiplier for the distribution center

The algorithm must also incorporate time urgency weighting, where the urgency multiplier is **75%** of the maximum time window span across all destinations. The maximum time window span is calculated as: `max(latest_time - earliest_time for earliest_time, latest_time in time_windows)`.

**Urgency Weighting Formula**:
- `urgency_multiplier = 0.75 * max_time_window_span`
- `urgency_weight = urgency_multiplier / (priority_level * 10.0)`
- `weighted_cost = total_route_cost - urgency_weight`

The greedy algorithm selects assignments based on the `weighted_cost`, where higher priority destinations (lower priority_level numbers) receive more favorable urgency weighting by having their weighted costs reduced more significantly.

## Implementation Guidelines

1. **Greedy Selection Strategy**: At each iteration, select the route assignment with the lowest `weighted_cost` (total_route_cost - urgency_weight) among all feasible options
2. **Capacity Tracking**: Maintain running totals of assigned demand for each distribution center and check capacity constraints before assignment
3. **Time Window Scheduling**: Schedule deliveries at the midpoint of each destination's time window: `(earliest_time + latest_time) // 2`
4. **Priority Consideration**: When multiple routes have identical weighted costs, prioritize destinations with higher priority levels (lower priority_level numbers)
5. **Urgency Weighting**: Apply urgency weighting where `urgency_weight = urgency_multiplier / (priority_level * 10.0)` and subtract from total_route_cost to favor higher priority destinations
6. **Deterministic Output**: The algorithm must produce consistent results for identical inputs
7. **Feasibility Check**: Before making any assignment, verify that the destination's demand doesn't exceed the distribution center's remaining capacity

## Algorithm Flow

1. Calculate maximum time window span for urgency weighting: `max_time_span = max(latest_time - earliest_time for earliest_time, latest_time in time_windows)`
2. Calculate urgency multiplier: `urgency_multiplier = 0.75 * max_time_span`
3. For each unassigned destination, calculate weighted_cost for each feasible distribution center using: `weighted_cost = total_route_cost - (urgency_multiplier / (priority_level * 10.0))`
4. Select the assignment with the minimum weighted_cost
5. Update capacity tracking and mark the destination as assigned
6. Repeat until all destinations are assigned or no feasible assignments remain

The algorithm should terminate successfully when all destinations are assigned within their respective constraints, or identify unassigned destinations if capacity limitations prevent complete assignment.
