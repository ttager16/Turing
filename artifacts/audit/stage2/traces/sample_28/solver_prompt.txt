Problem Statement:
### Problem Statement

A logistics company operates a multi-tier supply chain network where products flow from suppliers through intermediate warehouses to distribution centers across multiple shipping phases. Your task is to implement a brute force simulation that systematically evaluates all available routing options at each decision point. The simulation uses exhaustive local enumeration: at each step, check every possible route combination and select the best option. The system must handle coalition partnerships between suppliers, dynamic network events that modify routing options, warehouse capacity constraints, and multi-criteria cost optimization.

Each shipping phase represents a discrete time window where suppliers ship inventory to warehouses, which then forward shipments to distribution centers. The brute force simulation means: (1) at each decision point, exhaustively enumerate all feasible supplier-warehouse-DC combinations, (2) evaluate cost efficiency for every option, (3) select and commit to the best route, (4) update resources, and (5) repeat. Coalitions between suppliers provide cost and time discounts. The simulation terminates when demand is satisfied, resources are exhausted, or no feasible routes remain. This brute force approach guarantees locally optimal decisions through complete enumeration at each step.

---

#### Objectives:

1. Multi-Phase Brute Force Simulation: Implement brute force simulation for routing decisions across shipping phases (1 to 4 phases). Within each phase, use exhaustive local enumeration: at each decision point, evaluate ALL feasible routes by checking every supplier-to-warehouse and warehouse-to-distribution-center path combination. This complete enumeration compares costs across every possible supplier-warehouse-DC triple. Select the route with best cost-per-unit efficiency, commit to shipping maximum feasible units, update resources, and repeat. The brute force nature means exhaustive evaluation at each step. Termination conditions: (a) all demand satisfied, (b) no supplier inventory remains, (c) no feasible routes exist, or (d) maximum iterations reached (bounded by total demand units).

2. Coalition Handling: Process supplier coalitions that provide cost reductions of 15 percent and time improvements of 20 percent when coalition members use shared routes. Handle coalition formation and dissolution events that modify coalition structure between phases. Apply discounts only when supplier is part of a coalition.

3. Event-Driven Network Updates: Apply phase-specific events that modify network state including route closures (mark specific routes unavailable for subsequent phases) and capacity changes (modify warehouse throughput limits for subsequent phases). Events apply at start of specified phase and persist unless superseded.

4. Capacity-Constrained Routing: Respect warehouse capacity limits that restrict total units flowing through each warehouse per phase. Ensure supplier inventory limits are not exceeded in each phase. Track deliveries to distribution centers to satisfy demand when possible.

5. Multi-Criteria Optimization: Primary objective is minimize total cost across all phases and routes (including coalition discounts). Secondary objective is minimize total time. Calculate unmet demand for distribution centers when inventory or capacity constraints prevent full satisfaction.

6. Brute Force Enumeration with Termination: At each decision point, perform brute force enumeration of ALL available routing options - check every supplier-warehouse-DC combination systematically. This exhaustive evaluation compares cost-per-unit efficiency across the complete set of feasible routes. Select the best option, commit resources, and iterate. Termination safeguards prevent infinite loops: (1) track remaining demand/inventory, (2) break when no feasible routes found (best_route == None), (3) implicit bound on iterations (cannot exceed sum of all demand values), and (4) monotonic decrease in available resources ensures convergence.

- Brute force means complete local enumeration: at each step, check ALL S×W×DC combinations before selecting best
- Termination guaranteed by: (1) explicit check for no feasible routes, (2) monotonic resource depletion, (3) bounded iterations
- No global backtracking - decisions are committed sequentially
- Return empty dictionary {} if no valid routing exists that respects all capacity constraints
- Return dictionary with unmet_demand showing shortfalls if inventory insufficient to meet all demand
- Coalition benefits only apply when coalition exists and route is used by coalition member
- Warehouse capacity applies to throughput (units flowing through in that phase)
- Must work with Python standard library only
