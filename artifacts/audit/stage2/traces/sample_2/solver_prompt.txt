# Problem Statement

Design an advanced 2-SAT analysis system that solves Boolean satisfiability problems with exactly two literals per clause and performs comprehensive structural analysis of the solution space. Given variables numbered from 1 to num_variables and a list of clauses where each clause contains exactly two literals, construct a directed implication graph with 2×num_variables nodes representing each literal and its negation. Each clause (a OR b) generates two directed edges: NOT a -> b and NOT b -> a.

Implement Kosaraju's algorithm to identify strongly connected components through two depth-first searches. The first DFS on the original graph records finish times in a stack. The second DFS on the transposed graph processes nodes in decreasing finish order and assigns SCC indices starting from 0 for the first component discovered. The instance is unsatisfiable if any variable has both literals in the same SCC.

For satisfiable instances, compute the lexicographically smallest satisfying assignment by processing variables in order from 1 to num_variables. For each variable x, if its positive literal has a higher SCC index than its negative literal, assign x to true, otherwise assign x to false. This produces a deterministic assignment based on the topological ordering of SCCs.

Calculate the backbone by identifying forced literals that must have the same truth value in all satisfying assignments. A literal is forced true if assigning it false makes the formula unsatisfiable. To detect forced literals, for each unforced variable in the initial solution, temporarily add a unit clause forcing that variable to the opposite value, rebuild the implication graph, rerun SCC detection, and check for contradictions. Variables that cause contradictions when flipped are backbone variables.

Compute the implication closure for specified query literals by performing depth-first traversal from each query literal in the implication graph and collecting all reachable literals. The closure represents all literals that must be true whenever the query literal is true. Sort the closure results in ascending order by absolute value, with negative literals before positive for the same variable.

Analyze equivalence classes by identifying variable pairs where both variables always have the same truth value in all solutions. Two variables x and y are equivalent if x->y and y->x in the implication graph, which occurs when they belong to the same SCC. Similarly, identify opposition classes where variable pairs always have opposite truth values, which occurs when x and -y are in the same SCC.

For unsatisfiable instances, identify all variables whose positive and negative literals share the same SCC and report them as conflicting variables sorted in ascending order. Extract a minimal conflict core by performing breadth-first search from the positive literal of the first conflicting variable to its negative literal, collecting all edges traversed, then mapping those edges back to their originating clauses and including only unique clauses in the core.

Compute graph statistics including total nodes (always 2×num_variables), total directed edges (always 2×number of clauses), number of SCCs, size of the largest SCC, and average SCC size. For each SCC, calculate its condensation graph out-degree by counting edges to nodes in different SCCs. Identify sink SCCs with out-degree zero.

**If any constraint is violated, return an error dictionary with format {"error": "error_message"} where error_message describes the specific validation failure. Do not raise exceptions.**
