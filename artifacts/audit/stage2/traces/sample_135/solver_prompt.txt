## Problem Statement

A smart city's emergency response system requires an advanced traffic management algorithm to determine optimal routes for emergency vehicles navigating through dynamic urban environments. The system must handle real-time traffic data where road conditions and travel times are uncertain due to unpredictable factors such as weather, accidents, construction, and varying traffic density.

You are tasked with implementing a sophisticated shortest path algorithm that operates on graphs with uncertain edge weights, where each road segment has multiple possible travel time scenarios with associated probability distributions. The algorithm must calculate the most reliable shortest path, considering uncertainty in travel times while ensuring emergency vehicles reach their destinations optimally under changing conditions.

## Core Requirements

Your solution must implement a **pure Python** algorithm (no external libraries) that processes uncertain graph data and generates the most reliable shortest path considering probability-weighted scenarios. The algorithm should handle multiple uncertainty scenarios for each edge and compute paths that minimize expected travel time while maintaining reliability.

### Key Operational Constraints

1. **Uncertainty Modeling**: Each edge has multiple travel time scenarios with associated probabilities that sum to 1.0
2. **Path Reliability**: The algorithm must consider both expected travel time and path reliability across all scenarios
3. **Graph Connectivity**: Handle disconnected components and unreachable destinations gracefully

### Advanced Algorithmic Requirements

The shortest path calculation must use an **expected value optimization** approach with reliability weighting:

**Expected Travel Time = Σ(scenario_probability × scenario_travel_time) for all scenarios**

**Reliability Score = Σ(scenario_probability) for all scenarios where travel_time ≤ threshold, otherwise 1.0 if no scenarios qualify**

Where the reliability threshold is calculated as: **threshold = 1.25 × expected_time** for each edge.

The final path selection criterion combines both factors:

**Path Score = Expected_Total_Time + (Reliability_Penalty × (1 - Path_Reliability))**

Where:
- Expected_Total_Time: Sum of expected travel times along the path
- Path_Reliability: Product of individual edge reliability scores along the path
- Reliability_Penalty: **50.0** (fixed penalty factor for unreliable paths)

The algorithm must also incorporate **scenario consistency checking** where paths are validated during the pathfinding process to ensure no scenario combination results in travel time exceeding **200%** of the expected path time. This validation checks all possible combinations of edge scenarios along the path to verify that the sum of travel times in any combination does not exceed the 200% threshold. Only paths that pass this validation should be considered as valid candidates during the shortest path search.

## Implementation Guidelines

1. **Expected Value Calculation**: For each edge, compute expected travel time as the probability-weighted sum of all scenario travel times
2. **Reliability Computation**: Calculate reliability threshold as 1.25 × expected_time for each edge, then compute reliability as the sum of probabilities for scenarios where travel_time ≤ threshold (use 1.0 if no scenarios qualify)
3. **Path Optimization**: Use modified Dijkstra's algorithm where the distance metric is the Path Score (expected time + reliability penalty)
4. **Scenario Consistency**: During pathfinding, validate each candidate path to ensure no scenario combination exceeds 200% of expected path time by checking all possible combinations of edge scenarios along the path; only consider paths that pass this validation
5. **Variance Calculation**: Compute travel time variance for the selected path across all scenario combinations
6. **Deterministic Selection**: When multiple paths have identical scores, select the path with the lexicographically smallest node sequence
7. **Graph Representation**: Build an adjacency list representation for efficient pathfinding operations

## Algorithm Flow

1. **Preprocessing**: Build graph adjacency list and calculate expected times and reliability scores for all edges
2. **Reliability Threshold**: For each edge, calculate threshold = 1.25 × expected_time and determine reliability score
3. **Modified Dijkstra**: Apply the shortest path algorithm using Path Score as the distance metric, validating scenario consistency for each path during exploration
4. **Path Validation**: During pathfinding, check each candidate path to verify scenario consistency by testing all possible combinations of edge scenarios to ensure no combination exceeds 200% of expected path time before considering it as a valid option
5. **Scenario Analysis**: Calculate best-case, worst-case, and variance metrics for the selected path
6. **Result Assembly**: Construct output dictionary with complete path information and reliability metrics
