Problem Statement:
### Problem Statement

A renowned theater consortium is organizing a **multi-production drama festival** where several plays are staged **simultaneously** across different venues. Each play consists of **multiple scenes**, and each scene requires **specific roles** to be filled by available actors. Actors may participate in **different plays**, but cannot perform in **overlapping or consecutive scenes** without enough buffer time between them.

Due to **dynamic script changes**, **varying travel times** between rehearsal venues, and **uncertain actor availability**, the organizers must build a **smart rehearsal scheduling system** that assigns actors to scenes efficiently, ensuring no conflicts, minimizing travel fatigue, and adapting to unexpected changes.

Your task is to **design a scheduling optimization function** that generates the most feasible rehearsal plan by analyzing role requirements, actor-role suitability, availability probabilities, and travel distances — while ensuring the overall schedule remains conflict-free and balanced.

---

### Objectives

1. **Integrated Availability and Overlap Mapping:**
   Construct a **bipartite mapping** between **actors** and **scenes/roles** based on rehearsal slots, ensuring no actor is double-booked or scheduled in overlapping or consecutive scenes without buffer time.

   * Build adjacency connections: **actors <--> (scene, role)** considering both **time feasibility** and **role eligibility**.
   * Enforce **buffer intervals** between any two scenes involving the same actor.

2. **Probability-Backed Role Allocation:**
   Use **probabilistic availability modeling** based on historical attendance, travel fatigue, and role preference to compute each actor’s likelihood of being available for a scene.

3. **Compound Scheduling Optimization:**
   Merge **availability**, **role suitability**, and **travel times** into a **cost optimization framework** that produces the best-balanced rehearsal plan.

   * Minimize **travel exhaustion** using transport times between actor rehearsal locations.
   * Balance workloads so that no actor is overloaded with consecutive scenes.
   * Ensure all **role requirements** per scene are fulfilled within available rehearsal slots.

---

### Optimization Metric Definition

The optimization score for assigning an actor to a specific scene-role pair is defined as:

[
\text{Score(actor, scene, role)} = 2.0 \times P_{\text{available}} + 1.5 \times W_{\text{role-fit}} - 0.3 \times U_{\text{workload}} - 0.1 \times C_{\text{travel}} - P_{\text{name}}
]

Where:

| Symbol                 | Definition                                                                                                         |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------ |
| (P_{\text{available}}) | Probability of the actor being available at that time slot, derived from historical attendance.                    |
| (W_{\text{role-fit}})  | Role suitability weight between actor and role (from `role_mapping` input). Defaults to `1.0` if not provided.     |
| (U_{\text{workload}})  | Current workload = (assigned scenes ÷ max rehearsal capacity).                                                     |
| (C_{\text{travel}})    | Average travel cost among all actors assigned to the same scene.                                                   |
| (P_{\text{name}})      | Tie-breaker penalty defined as `0.001 × index_of_actor_in_sorted_list_by_name`. Used only for consistent ordering. |

The algorithm **maximizes this score** for all valid `(actor, scene, role)` assignments.

---

### Constraints

1. Each scene requires one or more **roles** to be filled; an actor can fill **only one role per scene**.
2. An actor cannot appear in **two concurrent** or **back-to-back** scenes without a **minimum buffer interval**.
3. Each actor has a **rehearsal capacity limit** (maximum number of scenes they can handle).
4. **Availability probability** must be computed from historical attendance records.
5. Only **affected scenes** should be recalculated if any scene is merged, split, or changed.
6. Input and output must use **lists** and **dictionaries** only.
7. Every scene must include **time slot information** to verify overlap and buffer feasibility.

### Key Definitions

* **Actor Availability:**
  A list of integer time slots during which an actor can rehearse.

* **Scene Requirements:**
  A list of dictionaries where each entry defines:

  * Scene ID
  * Time slot of the scene
  * Dictionary of roles and number of actors needed per role.

* **Historical Data:**
  Maps each actor to their attendance history (1 for attended, 0 for missed).

* **Role Mapping (W_role-fit):**
  Defines how suitable each actor is for each role, with values between 0 and 1.
  If not provided, all roles are assumed equally suitable (weight = 1.0).

* **Transport Times:**
  Travel times between rehearsal locations of different actors.

* **Buffer Interval:**
  Minimum time gap required between any two scenes involving the same actor.

* **Rehearsal Capacity:**
  Maximum number of scenes an actor can participate in.

### Input Format

1. **actor_availability** – List of lists, where each sub-list contains `[actor_name, [available_time_slots]]`.
   Example: `[['Actor1', [1, 2, 3, 5]], ['Actor2', [1, 3, 4, 6]]]`

2. **scene_requirements** – List of dictionaries, where each dictionary specifies:

   * `"scene_id"`: Unique string ID (e.g., `"Scene1"`)
   * `"time_slot"`: Integer rehearsal time slot
   * `"roles"`: Dictionary with roles and number of required actors
     Example: `{"scene_id": "Scene1", "time_slot": 2, "roles": {"role1": 2, "role2": 1}}`

3. **historical_data** – Dictionary mapping actors to lists of integers (1 = attended, 0 = missed).
   Example: `{'Actor1': [1, 0, 1], 'Actor2': [1, 1, 1]}`

4. **role_mapping** – Dictionary mapping each actor to another dictionary of role suitability scores.
   Example: `{'Actor1': {'role1': 0.9, 'role2': 0.6}, 'Actor2': {'role1': 0.7, 'role2': 1.0}}`

5. **transport_times** – Dictionary defining travel times between actors’ rehearsal locations.
   Example: `{'Actor1-Actor2': 3, 'Actor2-Actor3': 4}`

6. **buffer_intervals** – Integer defining the minimum time slot gap between two scenes involving the same actor.

7. **rehearsal_capacity** – Dictionary mapping each actor to their maximum number of scenes.
   Example: `{'Actor1': 3, 'Actor2': 2, 'Actor3': 3}`
