# Dynamic BFS Urban Delivery Network Optimization

## Problem Statement

A transportation logistics company operates in a complex urban environment where traffic conditions dynamically affect road availability and traversal costs. The company's delivery network is modeled as a dynamic graph where intersections represent nodes and roads represent edges that can become active or inactive based on real-time traffic data and time-dependent constraints.

You are tasked with implementing a sophisticated Dynamic Breadth-First Search (BFS) algorithm that navigates through this time-sensitive graph structure. The system must efficiently find optimal delivery paths while considering traffic activation patterns, delivery time windows, and capacity constraints across multiple priority levels.

## Core Requirements

Your solution must implement a **pure Python** Dynamic BFS algorithm (no external libraries) that processes urban delivery network data and generates optimal routing plans. The algorithm should prioritize paths based on delivery efficiency while respecting all operational and temporal constraints.

### Key Operational Constraints

1. **Traffic Activation Constraints**: Roads (edges) activate and deactivate based on time intervals and traffic flow capacity
2. **Time Window Constraints**: Each delivery location must be reached within specified time windows
3. **Capacity Constraints**: Each road segment has maximum vehicle capacity limits
4. **Priority Level Constraints**: Higher priority deliveries (lower priority numbers) must be processed first
5. **Path Continuity**: The algorithm must ensure continuous path availability throughout the delivery timeframe

### Advanced Algorithmic Requirements

The Dynamic BFS must incorporate a multi-layered scoring system that evaluates path viability using this composite formula:

**Path Efficiency Score = Base Travel Cost + (Time Penalty × Window Urgency) + (Capacity Utilization Factor × Priority Weight)**

Where:
- Base Travel Cost: Sum of edge weights along the path
- Time Penalty: Calculated as (current_time - latest_delivery_time) when current_time > latest_delivery_time, otherwise 0
- Window Urgency: Calculated as 100 / (latest_delivery_time - earliest_delivery_time + 1)
- Capacity Utilization Factor: (vehicle_load / road_capacity) × 50
- Priority Weight: priority_level × 25

The algorithm must also implement traffic flow prediction where roads with higher traffic density (traffic_density > 0.7) have their activation windows reduced by **20%** of their original duration.

**Traffic Adjustment Formula**:
- If traffic_density > 0.7: adjusted_duration = original_duration × 0.8
- Otherwise: adjusted_duration = original_duration

## Implementation Guidelines

**Dynamic BFS Strategy**: Implement time-aware BFS that considers road activation windows and traffic conditions at each time step
**Priority Processing**: Process delivery locations in priority order (priority_level 1 first, then 2, etc.)
**Traffic Adjustment**: Apply traffic density adjustments to activation windows before path calculation
**Path Efficiency Calculation**: Calculate path efficiency scores using the composite formula for each potential route
**Capacity Tracking**: Monitor road segment utilization and identify bottlenecks where capacity_utilization > 0.8
**Time Window Scheduling**: Schedule deliveries at the earliest feasible time within each location's time window
**Weather Impact**: Apply weather_impact_factor to all travel times during path calculation
**Deterministic Output**: The algorithm must produce consistent results for identical inputs
**Construction Zone Handling**: Reduce capacity by 50% for road segments connected to construction zone nodes

## Algorithm Flow

1. **Preprocessing Phase**: Calculate adjusted activation windows for high-traffic roads and apply weather impact factors to travel times
2. **Priority Sorting**: Sort delivery_locations by priority_level in ascending order
3. **Dynamic BFS Execution**: For each delivery location, perform BFS considering time-dependent road availability
4. **Path Evaluation**: Calculate path efficiency scores for all feasible routes and select optimal paths
5. **Capacity Analysis**: Track road segment utilization and identify bottleneck segments
6. **Schedule Optimization**: Assign delivery times within time windows while minimizing the total system efficiency score
7. **Result Compilation**: Generate a comprehensive output, including traffic utilization summary

The algorithm should terminate successfully when all deliverable locations are processed within their constraints, or identify undeliverable locations when no feasible paths exist within the given time and capacity limitations.
