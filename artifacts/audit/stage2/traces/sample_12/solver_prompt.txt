Problem Statement:
### PROBLEM STATEMENT

EA’s distributed multiplayer platform dynamically distributes players across several global regions. Each region hosts **alliances** of players, subject to both **local capacity limits** and **network synergy constraints** between neighboring regions.

Each pair of neighboring regions has a **synergy limit**, which restricts the total number of their combined **regional alliances** to prevent traffic overload. When this limit is exceeded, the platform creates **cross-region alliances** that merge players from both regions.  

Cross-region alliances:
- Operate at **50% of the smaller region's capacity**.
- **Do not count** toward synergy constraints.

However, such reallocation changes regional alliance counts and can indirectly trigger new violations in other regions. The system therefore rebalances repeatedly until **all constraints are satisfied** or the configuration becomes **infeasible**.

The goal is to compute the **minimum total number of alliances (regional + cross-region)** required to accommodate all players while satisfying every capacity and synergy constraint.  
If no stable configuration exists, return **-1**.

***

### OBJECTIVES

1. Compute each region’s **initial number of regional alliances**.
2. Detect synergy violations and rebalance through **cross-region alliances**.
3. Repeat adjustments until all constraints are satisfied or the system stagnates.
4. Minimize the total number of alliances formed.
5. Return the **final alliance count**, or **-1** if no feasible arrangement exists.

***

### REBALANCING ALGORITHM

For each iteration:
1. Identify all **synergy violations** where $$ A_a + A_b > limit $$.
2. Sort violations in **descending order of excess** ($$ (A_a + A_b) - limit $$).
3. For each violation:
   - Reallocate players to **cross-region alliances** one step at a time.
   - Choose moves that **minimize total alliances** (regional + cross-region) while resolving or over-resolving the violation.
   - Recalculate all regional alliance counts before processing the next violation.
4. Continue iterations until:
   - No violation remains, or
   - No alliances change during a full iteration (stagnation).

All data structures must be **lists**.  
**Tuples, sets, or external libraries are not allowed.**

***

### FORMULAS AND CONSTRAINTS

- **Initial regional alliances**:
  $$
  A_i = \lceil \frac{players[i]}{constraints[i]} \rceil
  $$

- **Synergy constraint** for each pair [a, b, limit]:
  $$
  A_a + A_b \leq limit
  $$

- **Cross-region alliance capacity**:
  $$
  0.5 \times \min(constraints[a], constraints[b])
  $$

Cross-region alliances relieve both participating regions but are ignored in synergy checks.

***

### EDGE CASES

1. **Zero-Player Regions:**  
   If `players[i] = 0`, region i has no alliances but can still appear in synergy definitions.

2. **Disconnected Regions:**  
   Regions with no synergies are balanced independently.  
   Their alliance counts contribute individually to the global total.

3. **Zero Synergy Limit:**  
   If `synergy_limit = 0` and both regions have players, all players must join cross-region alliances.  
   If impossible, return **-1**.

4. **Infeasible Capacity:**  
   If a region’s players cannot be accommodated even after maximum cross-region formation, return **-1**.

5. **Termination Conditions:**  
   Iteration stops when:
   - All synergy constraints are met, or
   - No alliance count changes between iterations.

***

**Function Signature**

