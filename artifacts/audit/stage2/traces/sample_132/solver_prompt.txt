# Problem Statement

A city council is undertaking extensive urban infrastructure planning that requires analyzing the impact of temporarily closing multiple intersections due to construction projects. The city's road network is modeled as an undirected graph where intersections represent nodes and roads represent edges. The council needs to efficiently answer multiple offline queries about shortest paths and connectivity between neighborhoods when various combinations of intersections are simultaneously restricted.

You are tasked with implementing a sophisticated **Offline Graph Query System** using **pure Python** (no external libraries) that preprocesses the urban road network and efficiently answers batched queries about path optimization and connectivity analysis under different intersection closure scenarios.

## Core Requirements

Your solution must implement a comprehensive offline query processing system that handles multiple types of graph analysis queries on urban road networks. The system should preprocess the network data to enable efficient query responses while considering dynamic intersection closures and traffic flow optimization.

### Key Operational Constraints

1. **Intersection Closure Constraints**: Handle simultaneous closure of multiple intersections with different closure durations
2. **Query Batch Processing**: Process multiple queries efficiently using precomputed data structures
3. **Path Validity Constraints**: Ensure all returned paths avoid closed intersections during specified time periods
4. **Network Connectivity Constraints**: Maintain connectivity analysis between critical neighborhood clusters
5. **Distance Optimization**: Calculate shortest paths considering road segment weights and traffic conditions

### Advanced Algorithmic Requirements

The Offline Query System must incorporate a multi-dimensional scoring system for path evaluation using this composite formula:

**Path Quality Score = Base Distance Cost + (Closure Penalty × Affected Segments) + (Traffic Impact Factor × Congestion Level) + (Detour Complexity × Alternative Routes Count)**

Where:
- Base Distance Cost: Sum of segment_length values for all edges in the path
- Closure Penalty: Calculated as (number_of_closed_intersections_on_path × 100) + (total_closure_duration × 0.5)
- Affected Segments: Number of road segments in the current path that connect to any closed intersection
- Traffic Impact Factor: (average_traffic_density × 25) where average is calculated from all edges in the path
- Congestion Level: Sum of congestion_weight values for all edges in the path
- Detour Complexity: (current_path_length - direct_shortest_path_length) × 10 where direct_shortest_path_length is the shortest path without considering closures
- Alternative Routes Count: Total number of feasible alternative paths found between source and destination

The system must also implement **intersection criticality analysis** where intersections with degree > 4 (more than 4 connected roads) have their closure impact multiplied by **1.5** in all calculations.

**Criticality Adjustment Formula**:
- If intersection_degree > 4: impact_multiplier = 1.5
- Otherwise: impact_multiplier = 1.0
