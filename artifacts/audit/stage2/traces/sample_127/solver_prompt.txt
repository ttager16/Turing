### Problem Statement

In a next-generation multi-zone smart city infrastructure project, engineers must establish a multi-layered power grid that spans different terrains, ecological habitats, and technical restrictions simultaneously. The system must integrate dynamic constraints such as phased environmental regulation changes, cable capacity expansions, and safety compliance tiers that evolve over the project timeline.
Each edge in the system corresponds to a candidate cable route linking two zones and includes label functions that compute multiple cost factors based on external states such as time windows, partial assignments of other edges, or zone-specific rules.

The task is to engineer a Python-based solution that determines an optimal spanning structure minimizing the total weighted cost while enforcing all ecological, modular expansion, and operational safety constraints across multiple project stages.
The algorithm must ensure deterministic behavior, reproducibility, and correctness for all valid inputs.

### Objectives

1. **Advanced Cost Evaluation**
   Implement label functions that compute edge costs. Each label function must depend on a shared runtime state dictionary passed to it at execution.

2. **Algorithm for Interconnected Constraints**
   Implement a deterministic variant of Kruskal’s algorithm to compute the minimum spanning structure, ensuring that all constraints are enforced before adding each edge.

3. **Multi-Stage Optimization**
   Recompute the spanning structure when state variables such as phase index or load change. Earlier edge selections must be reevaluated when costs or connectivity conditions differ in later stages.

### Additional Constraints and Instructions

* The `graph` dictionary must include two mandatory special keys:

  * `"_state"` — a dictionary passed to every label function as its single argument.
  * `"_constraints"` — a dictionary specifying runtime control parameters.

* The `_constraints` dictionary must include:

  * `"required_edges"`: list of edges that must be included.

    * If these edges form a cycle, the function must return:

      ```python
      [{"error": "Required edges form a cycle"}]
      ```

  * `"forbidden_edges"`: list of edges that must be excluded.

    * If a required edge is also forbidden, return:

      ```python
      [{"error": "Required edge is forbidden"}]
      ```

  * `"degree_limits"`: dictionary `{node: max_degree}`.

    * If degree limits make the spanning structure impossible, return:

      ```python
      [{"error": "Degree limit infeasible"}]
      ```

  * `"phase_blocklist"`: list of edges to exclude for the current phase index.

  * `"round_cost_to"`: integer specifying the number of decimal places to round edge costs before comparison.

* All edges must be treated as **undirected** and represented in canonical form `(min(u, v), max(u, v))`.

* The algorithm must sort edges deterministically by `(rounded_cost, u, v)` and apply strict lexicographic tie-breaking.

* The result must be lexicographically sorted before returning.

* If the graph is disconnected, return a **minimum spanning forest** covering all string-labeled nodes.

* Each label function must be evaluated with `graph["_state"]` as input.

  * If the function raises an exception or returns a non-finite number, that edge must be skipped.
  * If all edges fail, return:

    ```python
    [{"error": "No valid edges available"}]
    ```

* Self-loops `(u == v)` are invalid and must be ignored.
